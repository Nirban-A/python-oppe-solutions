???+ abstract "тЪб Quick Reference"
    **Function signature:** `increment(d: dict, key: str, step: int, limit: int) -> None`

    **Core idea:** add `step` to `d[key]`, but cap the result at `limit`. Modify the dict **in-place** (no return value).

    ```python
    def increment(d, key, step, limit):
        d[key] = min(d[key] + step, limit)
    ```

    **Key rules:**
    - Modifies `d` in-place тАФ function returns `None`
    - `min(current + step, limit)` is the one-liner cap
    - If `d[key] + step > limit`, store `limit`; otherwise store `d[key] + step`

## Problem Statement

!!! note "Problem"
    Write a function `increment(d: dict, key: str, step: int, limit: int) -> None` that increments `d[key]` by `step`, but ensures the value never exceeds `limit`. The function modifies the dictionary in-place and returns `None`.

## Understanding the problem

There are two cases:

| Condition | What happens |
|-----------|-------------|
| `d[key] + step <= limit` | Store `d[key] + step` normally |
| `d[key] + step > limit` | Cap at `limit` |

The elegant way to express "whichever is smaller" is Python's built-in `min()`.

!!! insight "Key insight"
    `min(a, b)` returns the smaller of two values. `min(d[key] + step, limit)` automatically picks the capped value when the increment would overshoot тАФ no `if` statement needed.

---

## Step-by-step walkthrough

### Step 1 тАФ Understand in-place modification

The function has no `return` statement. It modifies the dictionary directly:

```python
d[key] = <new value>   # this changes the dict that was passed in
```

Because Python dicts are **mutable**, changes inside the function are visible outside.

### Step 2 тАФ Compute the new value with a cap

```python
new_value = d[key] + step          # naive increment
if new_value > limit:
    new_value = limit              # cap it
d[key] = new_value
```

### Step 3 тАФ Collapse to one line with min()

```python
d[key] = min(d[key] + step, limit)
```

These two are exactly equivalent тАФ the `min()` version is simply more Pythonic.

## Solution approaches

=== "Explanatory (if-else)"

    ```python
    def increment(d, key, step, limit):
        new_value = d[key] + step
        if new_value > limit:
            d[key] = limit
        else:
            d[key] = new_value
    ```

    Most readable for beginners тАФ every branch is explicit.

=== "Pythonic (min)"

    ```python
    def increment(d, key, step, limit):
        d[key] = min(d[key] + step, limit)
    ```

    One line. `min()` replaces the entire if-else. Preferred in interviews and for clean code.

=== "With ternary"

    ```python
    def increment(d, key, step, limit):
        candidate = d[key] + step
        d[key] = candidate if candidate <= limit else limit
    ```

    Explicit ternary тАФ fine, but `min()` is shorter and more idiomatic.

## Key takeaways

<div class="pb-cards">

<div class="pb-card">
<div class="pb-card-num">01</div>
<h4>min() is a natural cap</h4>
<p><code>min(value, ceiling)</code> is the idiomatic way to enforce an upper bound on a numeric value.</p>
</div>

<div class="pb-card">
<div class="pb-card-num">02</div>
<h4>In-place vs. return</h4>
<p>When a function modifies a mutable argument (dict, list) in-place, it should return <code>None</code>. Always check the expected return type.</p>
</div>

<div class="pb-card">
<div class="pb-card-num">03</div>
<h4>max() for a floor</h4>
<p>The mirror pattern: <code>max(value, floor)</code> enforces a <em>lower</em> bound. Combine both: <code>min(max(v, lo), hi)</code> to clamp a value in a range.</p>
</div>
