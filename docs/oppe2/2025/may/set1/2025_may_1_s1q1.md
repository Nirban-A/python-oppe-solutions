???+ abstract "⚡ Quick Reference"
    **Function:** `compute_electricity_bill(units: int) -> float`

    **Core idea:** three-slab billing — each slab has a per-unit rate and an optional fixed charge.

    ```python
    def compute_electricity_bill(units: int) -> float:
        if units <= 200:
            cost = units * 0.5
        elif units <= 400:
            cost = units * 0.75 + 150
        else:
            cost = units * 0.90 + 300
        return cost
    ```

    **Key rules:**
    - `≤ 200` units → `0.5` per unit, no fixed charge
    - `≤ 400` units → `0.75` per unit + `150` fixed
    - `> 400` units → `0.90` per unit + `300` fixed

---

## Problem Statement

!!! note "Problem"

!!! warning "Problem statement error"
    The problem states `compute_electricity_bill(210) -> 236.50`. This is incorrect — no formula consistent with the given rates produces 236.50 for 210 units. The flat-rate formula gives `210 × 0.75 + 150 = 307.50`. The correct answer for 210 units is **307.50**.

## Solution approaches

=== "Pythonic (if-elif-else)"

    ```python
    def compute_electricity_bill(units: int) -> float:
        if units <= 200:
            cost = units * 0.5
        elif units <= 400:
            cost = units * 0.75 + 150
        else:
            cost = units * 0.90 + 300
        return cost
    ```

=== "Explanatory (named rates)"

    ```python
    def compute_electricity_bill(units: int) -> float:
        if units <= 200:
            rate         = 0.5
            fixed_charge = 0
        elif units <= 400:
            rate         = 0.75
            fixed_charge = 150
        else:
            rate         = 0.90
            fixed_charge = 300
        return units * rate + fixed_charge
    ```

=== "Using lambda"

    ```python
    compute_electricity_bill = lambda units: (
        units * 0.5        if units <= 200 else
        units * 0.75 + 150 if units <= 400 else
        units * 0.90 + 300
    )
    ```

## Key takeaways

<div class="pb-cards">

<div class="pb-card">
<div class="pb-card-num">01</div>
<h4>if-elif chain — check lower slabs first</h4>
<p>Since <code>elif units <= 400</code> only runs when <code>units > 200</code> (the first condition already failed), the slabs don't overlap. Always order conditions from lowest to highest slab boundary.</p>
</div>

<div class="pb-card">
<div class="pb-card-num">02</div>
<h4>Flat rate vs tiered billing</h4>
<p>This problem uses a <em>flat rate</em> — the rate applies to <em>all</em> units, not just the units above the previous threshold. `210 × 0.75 + 150` charges 0.75 per unit for all 210 units. Tiered billing (like income tax) would only charge 0.75 on the extra 10 units.</p>
</div>

<div class="pb-card">
<div class="pb-card-num">03</div>
<h4>Return float — don't truncate</h4>
<p>The template's <code>return cost</code> suggests the result should be a float. Python arithmetic with <code>0.5</code>, <code>0.75</code>, <code>0.90</code> already produces floats — no explicit conversion needed.</p>
</div>
