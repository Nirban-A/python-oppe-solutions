???+ abstract "⚡ Quick Reference"
    **Function:** `is_reverse_combined_palindrome(s1: str, s2: str) -> bool`

    **Core idea:** reverse `s1`, concatenate with `s2`, check if the result is a palindrome.

    ```python
    def is_reverse_combined_palindrome(s1: str, s2: str) -> bool:
        combined = s1[::-1] + s2
        return combined == combined[::-1]
    ```

    **Key rules:**
    - Reverse `s1` using `s1[::-1]`
    - Concatenate: `reversed_s1 + s2`
    - Palindrome check: `combined == combined[::-1]`

---

## Problem Statement

!!! note "Problem"

## Tracing both examples

**Example 1:** `s1="mad"`, `s2="am"`
```
reversed s1 = "dam"
combined    = "dam" + "am" = "danam"
reversed    = "manad"
"danam" == "manad" → False ✓
```

**Example 2:** `s1="dam"`, `s2="am"`
```
reversed s1 = "mad"
combined    = "mad" + "am" = "madam"
reversed    = "madam"
"madam" == "madam" → True ✓
```

## Solution approaches

=== "Pythonic"

    ```python
    def is_reverse_combined_palindrome(s1: str, s2: str) -> bool:
        combined = s1[::-1] + s2
        return combined == combined[::-1]
    ```

=== "Explanatory"

    ```python
    def is_reverse_combined_palindrome(s1: str, s2: str) -> bool:
        reversed_s1 = s1[::-1]
        combined    = reversed_s1 + s2
        return combined == combined[::-1]
    ```

=== "Using lambda"

    ```python
    is_reverse_combined_palindrome = lambda s1, s2: (
        (lambda c: c == c[::-1])(s1[::-1] + s2)
    )
    ```

## Key takeaways

<div class="pb-cards">

<div class="pb-card">
<div class="pb-card-num">01</div>
<h4>s[::-1] reverses a string</h4>
<p>Slice with step <code>-1</code> returns a reversed copy without modifying the original. <code>s1[::-1]</code> reverses <code>s1</code>; applying it again to <code>combined</code> checks the palindrome - same operation, two uses.</p>
</div>

<div class="pb-card">
<div class="pb-card-num">02</div>
<h4>Order matters - reverse s1 first, then concatenate</h4>
<p>The operation is <code>reverse(s1) + s2</code>, not <code>reverse(s1 + s2)</code>. Only <code>s1</code> is reversed; <code>s2</code> is appended as-is.</p>
</div>

<div class="pb-card">
<div class="pb-card-num">03</div>
<h4>Return bool - not the string</h4>
<p>The template's return type hint says <code>str</code> but the examples return <code>True</code>/<code>False</code>. The function should return a <code>bool</code> - <code>combined == combined[::-1]</code> naturally produces one.</p>
</div>
