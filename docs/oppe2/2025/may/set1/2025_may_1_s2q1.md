???+ abstract "⚡ Quick Reference"
    **Function:** `is_arithmetic_progression(sequence: list) -> bool`

    **Core idea:** compute the common difference from the first two elements, then verify all consecutive pairs have the same difference.

    ```python
    def is_arithmetic_progression(sequence: list) -> bool:
        if len(sequence) <= 1:
            return True
        diff = sequence[1] - sequence[0]
        return all(sequence[i] - sequence[i-1] == diff
                   for i in range(2, len(sequence)))
    ```

    **Key rules:**
    - Sequences of length 0 or 1 are trivially AP
    - Compute `diff = sequence[1] - sequence[0]`
    - All consecutive differences must equal `diff`

## Solution approaches

=== "Pythonic (all + range)"

    ```python
    def is_arithmetic_progression(sequence: list) -> bool:
        if len(sequence) <= 1:
            return True
        diff = sequence[1] - sequence[0]
        return all(sequence[i] - sequence[i-1] == diff
                   for i in range(2, len(sequence)))
    ```

=== "Using zip for adjacent pairs"

    ```python
    def is_arithmetic_progression(sequence: list) -> bool:
        if len(sequence) <= 1:
            return True
        diffs = [b - a for a, b in zip(sequence, sequence[1:])]
        return len(set(diffs)) == 1
    ```

    `zip(sequence, sequence[1:])` generates all adjacent pairs. Computing their differences and checking if they all reduce to a single value confirms an AP.

=== "Explanatory (loop)"

    ```python
    def is_arithmetic_progression(sequence: list) -> bool:
        if len(sequence) <= 1:
            return True
        diff = sequence[1] - sequence[0]
        for i in range(2, len(sequence)):
            if sequence[i] - sequence[i-1] != diff:
                return False
        return True
    ```

=== "Using lambda"

    ```python
    is_arithmetic_progression = lambda seq: (
        len(seq) <= 1 or
        len(set(b - a for a, b in zip(seq, seq[1:]))) == 1
    )
    ```

## Key takeaways

<div class="pb-cards">

<div class="pb-card">
<div class="pb-card-num">01</div>
<h4>Compute diff from first pair, check all others</h4>
<p><code>diff = sequence[1] - sequence[0]</code> establishes the expected common difference. Then verify every subsequent pair matches. This is O(n) — one pass through the sequence.</p>
</div>

<div class="pb-card">
<div class="pb-card-num">02</div>
<h4>set(diffs) == 1 element — elegant check</h4>
<p>If all differences are the same, the set of differences has exactly one element. <code>len(set(diffs)) == 1</code> is a clean alternative that doesn't need to extract the first difference separately.</p>
</div>

<div class="pb-card">
<div class="pb-card-num">03</div>
<h4>Handle length 0 and 1 as edge cases</h4>
<p>A sequence with 0 or 1 element has no pairs to compare — it's trivially an AP. Without this guard, <code>sequence[1] - sequence[0]</code> would raise an <code>IndexError</code> on short sequences.</p>
</div>
