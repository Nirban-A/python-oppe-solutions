???+ abstract "⚡ Quick Reference"
    **Function:** `remove_second_and_second_last(s: str) -> str`

    **Core idea:** keep the first character, skip index 1, keep the middle, skip the second-last, keep the last.

    ```python
    def remove_second_and_second_last(s: str) -> str:
        return s[0] + s[2:-2] + s[-1]
    ```

    **Key rules:**
    - Remove index `1` (second character) and index `len(s)-2` (second-last character)
    - `s[0]` → first char | `s[2:-2]` → middle | `s[-1]` → last char
    - Guaranteed at least 4 characters - no edge case guard needed

---

## Problem Statement

!!! note "Problem"
    Write a function `remove_second_and_second_last(s)` that removes the character at index 1 and the character at index `len(s)-2` from the string.

## Tracing both examples

**`"abcdef"` (length 6):**
```
Index:  0  1  2  3  4  5
Char:   a  b  c  d  e  f
           ↑           ↑
        remove 1    remove 4 (len-2)

s[0]   = "a"
s[2:-2] = s[2:4] = "cd"
s[-1]  = "f"
→ "a" + "cd" + "f" = "acdf" ✓
```

**`"abcd"` (length 4):**
```
Index:  0  1  2  3
Char:   a  b  c  d
           ↑  ↑
        remove 1  remove 2 (len-2)

s[0]   = "a"
s[2:-2] = s[2:2] = ""  (empty - no middle)
s[-1]  = "d"
→ "a" + "" + "d" = "ad" ✓
```

!!! insight "s[2:-2] handles the middle cleanly"
    For length 4: `s[2:-2] = s[2:2] = ""` - empty string. For length 5: `s[2:-2] = s[2:3]` - one character. The same formula works for any length ≥ 4 with no special cases.

## Solution approaches

=== "Pythonic (slicing)"

    ```python
    def remove_second_and_second_last(s: str) -> str:
        return s[0] + s[2:-2] + s[-1]
    ```

=== "Explanatory (named parts)"

    ```python
    def remove_second_and_second_last(s: str) -> str:
        first  = s[0]
        middle = s[2:-2]    # everything except first, second, second-last, last
        last   = s[-1]
        return first + middle + last
    ```

=== "Using list + del"

    ```python
    def remove_second_and_second_last(s: str) -> str:
        lst = list(s)
        del lst[-2]    # remove second-last first (doesn't shift index 1)
        del lst[1]     # then remove second
        return "".join(lst)
    ```

    Delete the second-last first - removing it doesn't change the index of the second character. Reverse order of deletion avoids index-shift bugs.

=== "Using lambda"

    ```python
    remove_second_and_second_last = lambda s: s[0] + s[2:-2] + s[-1]
    ```

## Key takeaways

<div class="pb-cards">

<div class="pb-card">
<div class="pb-card-num">01</div>
<h4>s[0] + s[2:-2] + s[-1] - three-part split</h4>
<p>The string is split into four regions: index 0 (keep), index 1 (skip), indices 2 to -2 (keep), index -2 (skip), index -1 (keep). Three slice operations reconstruct the result.</p>
</div>

<div class="pb-card">
<div class="pb-card-num">02</div>
<h4>s[2:-2] is empty for length 4 - no special case</h4>
<p>For a 4-character string, <code>s[2:-2] = s[2:2] = ""</code>. Concatenating an empty string does nothing. The same formula handles all lengths ≥ 4 without any <code>if</code> statements.</p>
</div>

<div class="pb-card">
<div class="pb-card-num">03</div>
<h4>When deleting from a list, remove later indices first</h4>
<p>If using the <code>del</code> approach, always delete the higher index first. Deleting index 1 first shifts all subsequent indices by -1, so <code>-2</code> would point to a different element.</p>
</div>
