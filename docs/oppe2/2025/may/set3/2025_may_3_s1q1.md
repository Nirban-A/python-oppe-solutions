???+ abstract "⚡ Quick Reference"
    **Function:** `is_three_digit_and_digit_sum_divisible_by_k(n: int, k: int) -> bool`

    **Core idea:** check the range, sum the digits, check divisibility.

    ```python
    def is_three_digit_and_digit_sum_divisible_by_k(n: int, k: int) -> bool:
        if not (100 <= n <= 999):
            return False
        digit_sum = sum(int(d) for d in str(n))
        return digit_sum % k == 0
    ```

    **Key rules:**
    - Three-digit: `100 <= n <= 999`
    - Digit sum: sum all digits of `n`
    - Divisible by k: `digit_sum % k == 0`
    - Both conditions must hold

## Problem Statement

!!! note "Problem"
    Write a function `is_three_digit_and_digit_sum_divisible_by_k(n, k)` that returns `True` only if `n` is a three-digit number AND the sum of its digits is divisible by `k`.

## Solution approaches

=== "Pythonic (string digit sum)"

    ```python
    def is_three_digit_and_digit_sum_divisible_by_k(n: int, k: int) -> bool:
        if not (100 <= n <= 999):
            return False
        return sum(int(d) for d in str(n)) % k == 0
    ```

=== "Explanatory (arithmetic digit extraction)"

    ```python
    def is_three_digit_and_digit_sum_divisible_by_k(n: int, k: int) -> bool:
        if not (100 <= n <= 999):
            return False
        hundreds = n // 100
        tens     = (n // 10) % 10
        units    = n % 10
        digit_sum = hundreds + tens + units
        return digit_sum % k == 0
    ```

    Extracts each digit using integer arithmetic - no string conversion needed.

=== "Using divmod"

    ```python
    def is_three_digit_and_digit_sum_divisible_by_k(n: int, k: int) -> bool:
        if not (100 <= n <= 999):
            return False
        total, temp = 0, n
        while temp:
            temp, digit = divmod(temp, 10)
            total += digit
        return total % k == 0
    ```

=== "Using lambda"

    ```python
    is_three_digit_and_digit_sum_divisible_by_k = lambda n, k: (
        100 <= n <= 999 and sum(int(d) for d in str(n)) % k == 0
    )
    ```

## Key takeaways

<div class="pb-cards">

<div class="pb-card">
<div class="pb-card-num">01</div>
<h4>Check range before digit sum</h4>
<p><code>100 <= n <= 999</code> is a cheap O(1) check. Return <code>False</code> early so the digit-sum computation only runs for valid three-digit numbers - short-circuit logic at its simplest.</p>
</div>

<div class="pb-card">
<div class="pb-card-num">02</div>
<h4>sum(int(d) for d in str(n)) - clean digit sum</h4>
<p>Converting to a string gives access to each digit character; <code>int(d)</code> converts back to an integer for summing. Works for any number of digits without manual division.</p>
</div>

<div class="pb-card">
<div class="pb-card-num">03</div>
<h4>Arithmetic extraction for three digits: // and %</h4>
<p>For exactly three digits: <code>n//100</code> = hundreds, <code>(n//10)%10</code> = tens, <code>n%10</code> = units. Direct and fast - no string conversion needed when you know the exact digit count.</p>
</div>
