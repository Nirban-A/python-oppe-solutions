???+ abstract "⚡ Quick Reference"
    **Type:** Full I/O problem

    **Core idea:** filter words by minimum length and starting character, collect their last characters, print concatenated.

    ```python
    l, c = input().split()
    l = int(l)
    n = int(input())
    result = ""
    for _ in range(n):
        word = input()
        if len(word) >= l and word[0] == c:
            result += word[-1]
    print(result)
    ```

    **Key rules:**
    - Word qualifies if `len(word) >= l` AND `word[0] == c` (case-sensitive)
    - Collect `word[-1]` for each qualifying word in order
    - Print the concatenation - empty string if no words qualify (prints an empty line)

---

## Problem Statement

!!! note "Problem (I/O type)"
    Read minimum length `l` and starting character `c`. Read `n` words. Print the concatenation of the last characters of all words with `len >= l` and starting with `c`.

**Example:**

## Tracing the example

`l=3`, `c="a"`

| Word | `len >= 3`? | `word[0] == 'a'`? | Qualifies? | Last char |
|------|------------|------------------|-----------|-----------|
| `apple` | ✅ (5) | ✅ | ✅ | `e` |
| `ant` | ✅ (3) | ✅ | ✅ | `t` |
| `banana` | ✅ (6) | ❌ (starts with 'b') | ❌ | - |
| `anchor` | ✅ (6) | ✅ | ✅ | `r` |
| `cat` | ✅ (3) | ❌ (starts with 'c') | ❌ | - |

Result: `"e" + "t" + "r"` = `"etr"` ✓

## Solution approaches

=== "Pythonic"

    ```python
    l, c = input().split()
    l = int(l)
    n = int(input())

    result = "".join(
        input()
        for _ in range(n)
        if (word := input()) and len(word) >= l and word[0] == c
    )
    print(result)
    ```

    !!! warning "Walrus operator reads input twice"
        The above has a bug - `input()` is called twice per iteration (once in the generator expression and once in the condition). Use the explicit loop below instead.

=== "Pythonic (loop)"

    ```python
    l, c = input().split()
    l = int(l)
    n = int(input())
    result = ""

    for _ in range(n):
        word = input()
        if len(word) >= l and word[0] == c:
            result += word[-1]

    print(result)
    ```

=== "Explanatory"

    ```python
    l_str, c = input().split()
    l = int(l_str)
    n = int(input())

    last_chars = []
    for _ in range(n):
        word = input()
        length_ok  = len(word) >= l
        starts_ok  = word[0] == c
        if length_ok and starts_ok:
            last_chars.append(word[-1])

    print("".join(last_chars))
    ```

=== "Using list comprehension + read-all"

    ```python
    l, c = input().split()
    l = int(l)
    n = int(input())
    words = [input() for _ in range(n)]

    result = "".join(w[-1] for w in words if len(w) >= l and w[0] == c)
    print(result)
    ```

    Read all words first into a list, then filter in a generator expression. Cleaner separation of input and processing.

## Key takeaways

<div class="pb-cards">

<div class="pb-card">
<div class="pb-card-num">01</div>
<h4>Two conditions - length AND starting character</h4>
<p>Both <code>len(word) >= l</code> and <code>word[0] == c</code> must be true. Short-circuit <code>and</code> skips the character check if the word is already too short.</p>
</div>

<div class="pb-card">
<div class="pb-card-num">02</div>
<h4>Case-sensitive - c is used as-is</h4>
<p>The problem specifies case-sensitive matching. <code>word[0] == c</code> distinguishes <code>'a'</code> from <code>'A'</code>. No <code>.lower()</code> should be applied.</p>
</div>

<div class="pb-card">
<div class="pb-card-num">03</div>
<h4>print("") outputs an empty line</h4>
<p>When no words qualify, <code>result</code> is an empty string. <code>print("")</code> outputs a blank line - exactly the required output for the no-results case.</p>
</div>
