???+ abstract "⚡ Quick Reference"
    **Function:** `is_multiple_of_5_not_3(num: int) -> bool`

    **Core idea:** check both divisibility conditions in one expression.

    ```python
    def is_multiple_of_5_not_3(num):
        return num % 5 == 0 and num % 3 != 0
    ```

    **Key rules:**
    - Multiple of 5: `num % 5 == 0`
    - NOT a multiple of 3: `num % 3 != 0`
    - Both must hold - use `and`
    - Works for negative numbers too (Python `%` is always non-negative for positive divisors)

---

## Problem Statement

!!! note "Problem"

## Solution approaches

=== "Pythonic (direct return)"

    ```python
    def is_multiple_of_5_not_3(num):
        return num % 5 == 0 and num % 3 != 0
    ```

=== "Explanatory"

    ```python
    def is_multiple_of_5_not_3(num):
        is_mult_of_5 = num % 5 == 0
        is_mult_of_3 = num % 3 == 0
        return is_mult_of_5 and not is_mult_of_3
    ```

=== "Using lambda"

    ```python
    is_multiple_of_5_not_3 = lambda num: num % 5 == 0 and num % 3 != 0
    ```

## Key takeaways

<div class="pb-cards">

<div class="pb-card">
<div class="pb-card-num">01</div>
<h4>Two conditions combined with and</h4>
<p><code>num % 5 == 0 and num % 3 != 0</code> - both must be true. Short-circuit <code>and</code> skips the second check if the first already fails (not a multiple of 5).</p>
</div>

<div class="pb-card">
<div class="pb-card-num">02</div>
<h4>Works for negatives - Python % is non-negative</h4>
<p>In Python, <code>-25 % 5 == 0</code> and <code>-25 % 3 == 2</code>. The modulo operator always returns a non-negative result when the divisor is positive, so no special handling is needed for negative inputs.</p>
</div>

<div class="pb-card">
<div class="pb-card-num">03</div>
<h4>15 is the classic trap - multiple of both</h4>
<p>15 is divisible by both 5 and 3 (it's a multiple of 15). The <code>num % 3 != 0</code> condition correctly excludes it. Always check both conditions explicitly.</p>
</div>
