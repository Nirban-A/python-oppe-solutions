???+ abstract "⚡ Quick Reference"
    **Function:** `combine_edges(s: str) -> str`

    **Core idea:** take the first two and last two characters; return empty string if length < 4.

    ```python
    def combine_edges(s: str) -> str:
        if len(s) < 4:
            return ""
        return s[:2] + s[-2:]
    ```

    **Key rules:**
    - `len(s) < 4` → return `""`
    - First two: `s[:2]`
    - Last two: `s[-2:]`
    - Concatenate both

---

## Problem Statement

!!! note "Problem"
    Write a function `combine_edges(s)` that returns a string made of the first two and last two characters of `s`. Return `""` if `len(s) < 4`.

## Tracing examples

| Input | Length | `s[:2]` | `s[-2:]` | Result |
|-------|--------|---------|---------|--------|
| `"Programming"` | 11 | `"Pr"` | `"ng"` | `"Prng"` ✓ |
| `"abc"` | 3 | - | - | `""` ✓ |
| `"abcd"` | 4 | `"ab"` | `"cd"` | `"abcd"` ✓ |
| `"ab"` | 2 | - | - | `""` ✓ |

!!! insight "Length exactly 4 - slices overlap safely"
    For `"abcd"` (length 4): `s[:2] = "ab"` and `s[-2:] = "cd"`. No overlap - the result is the full string. For length 3: `s[:2] = "ab"` and `s[-2:] = "bc"` would overlap - but we return `""` before reaching that.

## Solution approaches

=== "Pythonic"

    ```python
    def combine_edges(s: str) -> str:
        if len(s) < 4:
            return ""
        return s[:2] + s[-2:]
    ```

=== "Explanatory"

    ```python
    def combine_edges(s: str) -> str:
        if len(s) < 4:
            return ""
        first_two = s[:2]
        last_two  = s[-2:]
        return first_two + last_two
    ```

=== "One-liner with ternary"

    ```python
    def combine_edges(s: str) -> str:
        return s[:2] + s[-2:] if len(s) >= 4 else ""
    ```

=== "Using lambda"

    ```python
    combine_edges = lambda s: s[:2] + s[-2:] if len(s) >= 4 else ""
    ```

## Key takeaways

<div class="pb-cards">

<div class="pb-card">
<div class="pb-card-num">01</div>
<h4>s[:2] and s[-2:] - clean edge slicing</h4>
<p><code>s[:2]</code> always gives the first two characters; <code>s[-2:]</code> always gives the last two. These work for any string of length ≥ 4 without any index arithmetic.</p>
</div>

<div class="pb-card">
<div class="pb-card-num">02</div>
<h4>Guard length < 4 before slicing</h4>
<p>Without the guard, <code>"ab"[:2] + "ab"[-2:]</code> = <code>"ab" + "ab"</code> = <code>"abab"</code> - wrong. For strings shorter than 4, slices overlap and produce incorrect results. Always check length first.</p>
</div>

<div class="pb-card">
<div class="pb-card-num">03</div>
<h4>Exactly length 4 → returns the full string</h4>
<p>For <code>"abcd"</code>: <code>s[:2]="ab"</code> and <code>s[-2:]="cd"</code> together give <code>"abcd"</code>. The slices partition the string without overlap, so the result is the original string.</p>
</div>
