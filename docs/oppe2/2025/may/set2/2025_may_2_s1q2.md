???+ abstract "⚡ Quick Reference"
    **Function:** `starts_with_greeting(s: str) -> bool`

    **Core idea:** check if the string starts with `"Hello "` or `"Hi "` (space included).

    ```python
    def starts_with_greeting(s):
        return s.startswith("Hello ") or s.startswith("Hi ")
    ```

    **Key rules:**
    - The space after the greeting is mandatory - `"HelloWorld"` is `False`
    - Case-sensitive - `"hello World"` would be `False`
    - `str.startswith()` accepts a tuple to check multiple prefixes at once

---

## Problem Statement

!!! note "Problem"
    Write a function `starts_with_greeting(s)` that returns `True` if `s` starts with `"Hello "` or `"Hi "` (including the trailing space).

## Solution approaches

=== "Pythonic (startswith with tuple)"

    ```python
    def starts_with_greeting(s):
        return s.startswith(("Hello ", "Hi "))
    ```

    `str.startswith()` accepts a **tuple of prefixes** - returns `True` if the string starts with any of them. The cleanest single-expression solution.

=== "Explanatory (two checks)"

    ```python
    def starts_with_greeting(s):
        return s.startswith("Hello ") or s.startswith("Hi ")
    ```

=== "Using lambda"

    ```python
    starts_with_greeting = lambda s: s.startswith(("Hello ", "Hi "))
    ```

## Key takeaways

<div class="pb-cards">

<div class="pb-card">
<div class="pb-card-num">01</div>
<h4>The space is part of the prefix</h4>
<p><code>"Hello "</code> includes a trailing space - this ensures <code>"HelloWorld"</code> correctly returns <code>False</code>. Without the space, any string starting with <code>"Hello"</code> would match.</p>
</div>

<div class="pb-card">
<div class="pb-card-num">02</div>
<h4>startswith() accepts a tuple</h4>
<p><code>s.startswith(("Hello ", "Hi "))</code> checks both prefixes in one call - cleaner than two separate <code>startswith()</code> calls joined with <code>or</code>. The same trick works with <code>endswith()</code>.</p>
</div>

<div class="pb-card">
<div class="pb-card-num">03</div>
<h4>Case-sensitive by design</h4>
<p><code>startswith()</code> is case-sensitive. <code>"hello World"</code> and <code>"HI there"</code> would both return <code>False</code>. The problem doesn't ask for case-insensitive matching, so no <code>.lower()</code> needed.</p>
</div>
