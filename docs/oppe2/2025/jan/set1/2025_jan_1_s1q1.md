???+ abstract "⚡ Quick Reference"
    **Function:** `is_multiple(a: int, b: int) -> bool`

    **Core idea:** check if `a % b == 0` or `b % a == 0`.

    ```python
    def is_multiple(a: int, b: int) -> bool:
        return a % b == 0 or b % a == 0
    ```

    **Key rules:**
    - `a % b == 0` → `a` is a multiple of `b`
    - `b % a == 0` → `b` is a multiple of `a`
    - Either condition makes the result `True`

---

## Problem Statement

!!! note "Problem"
    Write a function `is_multiple(a, b)` that returns `True` if either `a` is a multiple of `b` or `b` is a multiple of `a`.

## Solution approaches

=== "Pythonic (direct return)"

    ```python
    def is_multiple(a: int, b: int) -> bool:
        return a % b == 0 or b % a == 0
    ```

=== "Explanatory"

    ```python
    def is_multiple(a: int, b: int) -> bool:
        a_is_multiple_of_b = a % b == 0
        b_is_multiple_of_a = b % a == 0
        return a_is_multiple_of_b or b_is_multiple_of_a
    ```

=== "Using any()"

    ```python
    def is_multiple(a: int, b: int) -> bool:
        return any(x % y == 0 for x, y in [(a, b), (b, a)])
    ```

=== "Using lambda"

    ```python
    is_multiple = lambda a, b: a % b == 0 or b % a == 0
    ```

## Key takeaways

<div class="pb-cards">

<div class="pb-card">
<div class="pb-card-num">01</div>
<h4>x % y == 0 means x is a multiple of y</h4>
<p>If <code>x</code> divided by <code>y</code> leaves no remainder, then <code>x</code> is a multiple of <code>y</code>. Check both directions — <code>a % b</code> and <code>b % a</code> — to cover either case.</p>
</div>

<div class="pb-card">
<div class="pb-card-num">02</div>
<h4>Short-circuit or — stops at first True</h4>
<p><code>a % b == 0 or b % a == 0</code> — if the first condition is already <code>True</code>, Python skips evaluating the second. Efficient and clean.</p>
</div>

<div class="pb-card">
<div class="pb-card-num">03</div>
<h4>Symmetric relationship</h4>
<p>The result is the same regardless of argument order — <code>is_multiple(10, 5)</code> and <code>is_multiple(5, 10)</code> both return <code>True</code>. The <code>or</code> of both directions ensures this symmetry.</p>
</div>
