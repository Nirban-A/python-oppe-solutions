???+ abstract "⚡ Quick Reference"
    **Type:** Full I/O problem

    **Core idea:** read each row, map each value to `"@"` or `"*"` based on the threshold, print space-separated.

    ```python
    n, t = map(int, input().split())
    for _ in range(n):
        row = map(int, input().split())
        print(" ".join("@" if v >= t else "*" for v in row))
    ```

    **Key rules:**
    - `value >= t` → `"@"` | `value < t` → `"*"`
    - Elements separated by spaces, no leading/trailing space
    - `" ".join(...)` handles spacing cleanly

---

## Problem Statement

!!! note "Problem (I/O type)"
    Read an `n×n` grid and threshold value `t`. Replace each pixel with `@` if ≥ `t`, else `*`. Print the result.

## Solution approaches

=== "Pythonic (generator + join)"

    ```python
    n, t = map(int, input().split())
    for _ in range(n):
        row = map(int, input().split())
        print(" ".join("@" if v >= t else "*" for v in row))
    ```

=== "Explanatory"

    ```python
    n, t = map(int, input().split())
    for _ in range(n):
        values  = list(map(int, input().split()))
        symbols = []
        for v in values:
            if v >= t:
                symbols.append("@")
            else:
                symbols.append("*")
        print(" ".join(symbols))
    ```

=== "Using list comprehension"

    ```python
    n, t = map(int, input().split())
    for _ in range(n):
        row = [int(x) for x in input().split()]
        result = ["@" if v >= t else "*" for v in row]
        print(" ".join(result))
    ```

=== "Using lambda + map"

    ```python
    n, t = map(int, input().split())
    threshold = lambda v: "@" if v >= t else "*"
    for _ in range(n):
        row = map(int, input().split())
        print(" ".join(map(threshold, row)))
    ```

    `map(threshold, row)` applies the lambda to every pixel. `" ".join(...)` formats the output. Clean one-liner per row.

## Key takeaways

<div class="pb-cards">

<div class="pb-card">
<div class="pb-card-num">01</div>
<h4>" ".join() — no leading or trailing space</h4>
<p><code>" ".join(symbols)</code> places exactly one space between elements and nothing at the ends. Using <code>print(*symbols)</code> also works — it joins with a space by default. Both avoid the trailing space issue.</p>
</div>

<div class="pb-card">
<div class="pb-card-num">02</div>
<h4>Process row by row — no need to store the full grid</h4>
<p>Read one row, convert, print — then move to the next. The grid never needs to be stored in memory all at once, making this efficient for large inputs.</p>
</div>

<div class="pb-card">
<div class="pb-card-num">03</div>
<h4>≥ t maps to "@" — inclusive threshold</h4>
<p>Values exactly equal to <code>t</code> map to <code>"@"</code> (not <code>"*"</code>). The condition is <code>v >= t</code>, not <code>v > t</code>. From example 1: value 5 with threshold 5 → <code>"@"</code> ✓</p>
</div>
