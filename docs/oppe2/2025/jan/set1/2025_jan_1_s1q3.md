???+ abstract "⚡ Quick Reference"
    **Function:** `add_middle_elems_to_ends_in_even_length_tuple(t: tuple) -> tuple`

    **Core idea:** find the two middle elements, prepend the first and append the second, keeping the rest in original order.

    ```python
    def add_middle_elems_to_ends_in_even_length_tuple(t: tuple) -> tuple:
        mid   = len(t) // 2
        mid1, mid2 = t[mid - 1], t[mid]
        return (mid1,) + t[:mid - 1] + t[mid + 1:] + (mid2,)
    ```

    **Key rules:**
    - For even-length tuple of length `n`: two middle elements are at indices `n//2 - 1` and `n//2`
    - `mid1` goes to position 0, `mid2` goes to the last position
    - The remaining elements keep their original relative order

---

## Problem Statement

!!! note "Problem"
    Write a function that extracts the two middle elements of an even-length tuple, places the first at the beginning and the second at the end, and returns the rearranged tuple.

## Understanding the structure

For `t = (10, 20, 30, 40)`, `len = 4`, `mid = 2`:

```
Index:     0    1    2    3
Value:    10   20   30   40
               ↑    ↑
              mid1 mid2  (indices mid-1=1 and mid=2)

Result structure:
(mid1,) + t[:mid-1]  + t[mid+1:]  + (mid2,)
  (20,) +   t[:1]    +   t[3:]    +  (30,)
  (20,) +   (10,)    +   (40,)    +  (30,)
= (20, 10, 40, 30) ✓
```

For `t = ("Lenovo", "Dell", "HP", "Asus")`, `mid = 2`:

```
mid1 = "Dell", mid2 = "HP"
t[:1] = ("Lenovo",), t[3:] = ("Asus",)
→ ("Dell", "Lenovo", "Asus", "HP") ✓
```

| | `t` | `mid` | `mid1` | `mid2` | `t[:mid-1]` | `t[mid+1:]` | Result |
|-|-----|-------|--------|--------|------------|------------|--------|

| Ex 2 | `("Lenovo","Dell","HP","Asus")` | 2 | `"Dell"` | `"HP"` | `("Lenovo",)` | `("Asus",)` | `("Dell","Lenovo","Asus","HP")` |

## Solution approaches

=== "Pythonic (slicing)"

    ```python
    def add_middle_elems_to_ends_in_even_length_tuple(t: tuple) -> tuple:
        mid  = len(t) // 2
        mid1 = t[mid - 1]
        mid2 = t[mid]
        return (mid1,) + t[:mid - 1] + t[mid + 1:] + (mid2,)
    ```

=== "Explanatory (named parts)"

    ```python
    def add_middle_elems_to_ends_in_even_length_tuple(t: tuple) -> tuple:
        n    = len(t)
        mid  = n // 2

        mid1 = t[mid - 1]   # first middle element
        mid2 = t[mid]        # second middle element

        left  = t[:mid - 1]  # elements before mid1
        right = t[mid + 1:]  # elements after mid2

        return (mid1,) + left + right + (mid2,)
    ```

=== "Using list unpacking"

    ```python
    def add_middle_elems_to_ends_in_even_length_tuple(t: tuple) -> tuple:
        mid  = len(t) // 2
        lst  = list(t)
        mid1 = lst.pop(mid - 1)     # removes and returns first middle
        mid2 = lst.pop(mid - 1)     # after pop, mid is now at mid-1 again
        return (mid1,) + tuple(lst) + (mid2,)
    ```

    After removing `mid1` with `pop(mid-1)`, the second middle element shifts to index `mid-1`. Two pops extract both middles cleanly; the remaining list becomes the inner part.

=== "Using lambda"

    ```python
    add_middle_elems_to_ends_in_even_length_tuple = lambda t: (
        (lambda m: (t[m-1],) + t[:m-1] + t[m+1:] + (t[m],))(len(t)//2)
    )
    ```

## Key takeaways

<div class="pb-cards">

<div class="pb-card">
<div class="pb-card-num">01</div>
<h4>Middle indices for even-length tuple: n//2-1 and n//2</h4>
<p>For a tuple of length 4: indices 1 and 2. For length 6: indices 2 and 3. The formula <code>mid = len(t)//2</code> gives the second middle; <code>mid-1</code> gives the first.</p>
</div>

<div class="pb-card">
<div class="pb-card-num">02</div>
<h4>Single-element tuples need a trailing comma</h4>
<p><code>(mid1,)</code> — the comma makes it a tuple. Without it, <code>(mid1)</code> is just parentheses around a value, not a tuple. Concatenating a non-tuple with a tuple raises a <code>TypeError</code>.</p>
</div>

<div class="pb-card">
<div class="pb-card-num">03</div>
<h4>pop() shifts indices — account for it</h4>
<p>After <code>lst.pop(mid-1)</code> removes the first middle element, the list is one shorter. The second middle element is now at index <code>mid-1</code> (not <code>mid</code>). The second <code>pop(mid-1)</code> correctly grabs it.</p>
</div>
