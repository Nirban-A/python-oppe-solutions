???+ abstract "⚡ Quick Reference"
    **Function:** `same_sign(a, b) -> bool`

    **Core idea:** extract the sign of each number as `-1`, `0`, or `1` using a helper, then compare.

    ```python
    def same_sign(a, b):
        sign = lambda x: 0 if x == 0 else (1 if x > 0 else -1)
        return sign(a) == sign(b)
    ```

    **Key rules:**
    - Three sign categories: positive (`> 0`), negative (`< 0`), zero (`== 0`)
    - `same_sign(0, 10)` → `False` - zero is its own category, not positive or negative
    - Both must be in the **same** category

---

## Problem Statement

!!! note "Problem"
    Write a function `same_sign(a, b)` that returns `True` if both numbers are strictly positive, both strictly negative, or both zero.

## Solution approaches

=== "Pythonic (sign helper)"

    ```python
    def same_sign(a, b):
        sign = lambda x: 0 if x == 0 else (1 if x > 0 else -1)
        return sign(a) == sign(b)
    ```

=== "Explanatory (three explicit cases)"

    ```python
    def same_sign(a, b):
        both_positive = a > 0 and b > 0
        both_negative = a < 0 and b < 0
        both_zero     = a == 0 and b == 0
        return both_positive or both_negative or both_zero
    ```

    Most readable - directly states each of the three valid cases. No helper function needed.

=== "Using math.copysign"

    ```python
    import math

    def same_sign(a, b):
        if a == 0 and b == 0:
            return True
        if a == 0 or b == 0:
            return False
        return math.copysign(1, a) == math.copysign(1, b)
    ```

    `math.copysign(1, x)` returns `1.0` if `x > 0` and `-1.0` if `x < 0`. Handle zero explicitly first since `copysign` treats `0` as positive.

=== "Using lambda"

    ```python
    same_sign = lambda a, b: (
        (a > 0 and b > 0) or (a < 0 and b < 0) or (a == 0 and b == 0)
    )
    ```

## Key takeaways

<div class="pb-cards">

<div class="pb-card">
<div class="pb-card-num">01</div>
<h4>Zero is its own sign category</h4>
<p>The problem treats zero separately from positive and negative. <code>same_sign(0, 10)</code> is <code>False</code> even though both might appear "non-negative". Always handle zero as a distinct third case.</p>
</div>

<div class="pb-card">
<div class="pb-card-num">02</div>
<h4>Three explicit cases - most readable</h4>
<p><code>both_positive or both_negative or both_zero</code> directly maps to the problem statement. Each condition is self-explanatory - no mental decoding of sign tricks needed.</p>
</div>

<div class="pb-card">
<div class="pb-card-num">03</div>
<h4>sign(a) == sign(b) - compact and extensible</h4>
<p>Mapping each number to its sign value (<code>-1</code>, <code>0</code>, <code>1</code>) and comparing is compact. Works for any number of arguments if extended to <code>len(set(map(sign, nums))) == 1</code>.</p>
</div>
