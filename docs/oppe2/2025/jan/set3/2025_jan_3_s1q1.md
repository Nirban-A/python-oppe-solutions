???+ abstract "⚡ Quick Reference"
    **Function:** `extract_border_elements(arr: list) -> list`

    **Core idea:** handle three cases - empty, single element, multiple elements.

    ```python
    def extract_border_elements(arr: list) -> list:
        if len(arr) == 0:
            return []
        if len(arr) == 1:
            return [arr[0]]
        return [arr[0], arr[-1]]
    ```

    **Key rules:**
    - Empty list → `[]`
    - Single element → `[arr[0]]`
    - Two or more elements → `[arr[0], arr[-1]]`

---

## Problem Statement

!!! note "Problem"
    Write a function `extract_border_elements(arr)` that returns a list with the first and last elements of the input list, handling edge cases for empty and single-element lists.

## Solution approaches

=== "Pythonic (if-elif-else)"

    ```python
    def extract_border_elements(arr: list) -> list:
        if len(arr) == 0:
            return []
        if len(arr) == 1:
            return [arr[0]]
        return [arr[0], arr[-1]]
    ```

=== "Compact (using slice trick)"

    ```python
    def extract_border_elements(arr: list) -> list:
        if not arr:
            return []
        return list({0: arr[0], -1: arr[-1]}.values()) if len(arr) > 1 else [arr[0]]
    ```

=== "Using set + sorted indices"

    ```python
    def extract_border_elements(arr: list) -> list:
        if not arr:
            return []
        if len(arr) == 1:
            return arr[:]
        return [arr[0], arr[-1]]
    ```

    `arr[:]` returns a shallow copy of the list - works cleanly for the single-element case.

=== "Using lambda"

    ```python
    extract_border_elements = lambda arr: (
        [] if not arr else
        [arr[0]] if len(arr) == 1 else
        [arr[0], arr[-1]]
    )
    ```

## Key takeaways

<div class="pb-cards">

<div class="pb-card">
<div class="pb-card-num">01</div>
<h4>Handle edge cases first</h4>
<p>Check empty and single-element cases before the general case. This avoids index errors and keeps the logic flat - no nested conditions needed.</p>
</div>

<div class="pb-card">
<div class="pb-card-num">02</div>
<h4>arr[-1] for the last element</h4>
<p>Negative indexing always gives the last element regardless of list length. <code>arr[-1]</code> is cleaner than <code>arr[len(arr)-1]</code> and works for any non-empty list.</p>
</div>

<div class="pb-card">
<div class="pb-card-num">03</div>
<h4>if not arr - Pythonic empty check</h4>
<p>Empty lists are falsy in Python. <code>if not arr</code> is the idiomatic way to check for an empty list - equivalent to <code>if len(arr) == 0</code> but shorter.</p>
</div>
