???+ abstract "⚡ Quick Reference"
    **Type:** Full I/O problem

    **Core idea:** read ball by ball, add runs per the rules, stop when out condition is met.

    ```python
    import sys

    balls = 0
    runs  = 0
    prev_bat = None

    for line in sys.stdin:
        line = line.strip()
        if not line:
            continue
        bat, bowl = map(int, line.split())
        balls += 1

        # Out conditions: same as bowler OR batsman plays same twice in a row
        if bat == bowl or bat == prev_bat:
            break

        runs += bowl if bat == 0 else bat
        prev_bat = bat

    print(balls, runs)
    ```

    **Key rules:**
    - Out if `bat == bowl` (same as bowler)
    - Out if `bat == prev_bat` (batsman plays same number twice consecutively)
    - If `bat == 0` → add bowler's number to runs
    - Otherwise → add batsman's number to runs
    - The ball that ends the game **counts** toward `balls` but **not** toward `runs`

## Solution approaches

=== "Pythonic (stdin loop)"

    ```python
    import sys

    balls    = 0
    runs     = 0
    prev_bat = None

    for line in sys.stdin:
        line = line.strip()
        if not line:
            continue
        bat, bowl = map(int, line.split())
        balls += 1

        if bat == bowl or bat == prev_bat:
            break

        runs    += bowl if bat == 0 else bat
        prev_bat = bat

    print(balls, runs)
    ```

=== "Using input() loop"

    ```python
    import sys

    balls    = 0
    runs     = 0
    prev_bat = None

    for line in sys.stdin:
        line = line.strip()
        if not line:
            continue
        bat, bowl = map(int, line.split())
        balls += 1

        # Check out conditions
        if bat == bowl:       # batsman plays same as bowler
            break
        if bat == prev_bat:   # batsman plays same number twice in a row
            break

        # Add runs
        if bat == 0:
            runs += bowl   # bowler's number counts
        else:
            runs += bat    # batsman's number counts

        prev_bat = bat

    print(balls, runs)
    ```

=== "Using itertools.takewhile (conceptual)"

    ```python
    import sys

    lines    = [l.strip() for l in sys.stdin if l.strip()]
    pairs    = [tuple(map(int, l.split())) for l in lines]

    prev_bat = None
    balls    = 0
    runs     = 0

    for bat, bowl in pairs:
        balls += 1
        if bat == bowl or bat == prev_bat:
            break
        runs    += bowl if bat == 0 else bat
        prev_bat = bat

    print(balls, runs)
    ```

## Key takeaways

<div class="pb-cards">

<div class="pb-card">
<div class="pb-card-num">01</div>
<h4>Two out conditions - bat==bowl OR bat==prev_bat</h4>
<p>The innings ends if the batsman plays the same number as the bowler <em>or</em> plays the same number as their previous ball. Track <code>prev_bat</code> across iterations and check both before adding runs.</p>
</div>

<div class="pb-card">
<div class="pb-card-num">02</div>
<h4>bat == 0 → use bowl's number</h4>
<p>When the batsman plays 0, the bowler's number is added instead. This is an exception to the normal rule - always check <code>if bat == 0</code> before adding runs.</p>
</div>

<div class="pb-card">
<div class="pb-card-num">03</div>
<h4>The out ball counts for balls but not for runs</h4>
<p>Increment <code>balls</code> before checking out conditions. Break immediately after incrementing - the out ball is counted but no runs are added for it.</p>
</div>
