???+ abstract "⚡ Quick Reference"
    **Function:** `merge_dictionaries(d1: dict, d2: dict) -> dict`

    **Core idea:** start with a copy of `d1`, then for each key in `d2` add its value to the existing value (or insert it if new).

    ```python
    def merge_dictionaries(d1: dict, d2: dict) -> dict:
        result = dict(d1)
        for key, value in d2.items():
            result[key] = result.get(key, 0) + value
        return result
    ```

    **Key rules:**
    - Keys unique to `d1` or `d2` → included as-is
    - Keys in both → values are **summed**
    - Original dicts must not be modified — work on a copy of `d1`

---

## Problem Statement

!!! note "Problem"
    Write a function `merge_dictionaries(d1, d2)` that merges two dictionaries, summing values for keys that appear in both.

**Example:**

<div class="pb-io">
<div class="pb-io-block input">
<span class="pb-io-label">Input</span>
<pre><code>d1={"a": 1, "b": 2}, d2={"b": 3, "c": 4}</code></pre>
</div>
<div class="pb-io-block output">
<span class="pb-io-label">Output</span>
<pre><code>{"a": 1, "b": 5, "c": 4}</code></pre>
</div>
</div>

`"b"` appears in both → `2 + 3 = 5`. `"a"` only in `d1` → 1. `"c"` only in `d2` → 4.

## Solution approaches

=== "Pythonic (dict.get + copy)"

    ```python
    def merge_dictionaries(d1: dict, d2: dict) -> dict:
        result = dict(d1)                          # copy d1
        for key, value in d2.items():
            result[key] = result.get(key, 0) + value
        return result
    ```

=== "Explanatory (explicit check)"

    ```python
    def merge_dictionaries(d1: dict, d2: dict) -> dict:
        result = dict(d1)    # start with all keys from d1
        for key, value in d2.items():
            if key in result:
                result[key] += value    # sum on conflict
            else:
                result[key] = value     # new key — insert as-is
        return result
    ```

=== "Using Counter"

    ```python
    from collections import Counter

    def merge_dictionaries(d1: dict, d2: dict) -> dict:
        return dict(Counter(d1) + Counter(d2))
    ```

    `Counter` supports `+` which sums values for matching keys and includes all unique keys. Clean one-liner — note that `Counter` drops zero/negative results, which is fine here since values are positive numbers.

=== "Using dict comprehension + set union"

    ```python
    def merge_dictionaries(d1: dict, d2: dict) -> dict:
        all_keys = d1.keys() | d2.keys()    # union of all keys
        return {k: d1.get(k, 0) + d2.get(k, 0) for k in all_keys}
    ```

    Takes the union of all keys, then for each key sums the value from `d1` (defaulting to 0) and `d2` (defaulting to 0). Elegant and symmetric — treats both dicts equally.

=== "Using lambda + reduce"

    ```python
    from functools import reduce

    def merge_dictionaries(d1: dict, d2: dict) -> dict:
        merge = lambda acc, kv: {**acc, kv[0]: acc.get(kv[0], 0) + kv[1]}
        return reduce(merge, d2.items(), dict(d1))
    ```

## Key takeaways

<div class="pb-cards">

<div class="pb-card">
<div class="pb-card-num">01</div>
<h4>Copy d1 first — don't modify the original</h4>
<p><code>result = dict(d1)</code> creates a shallow copy. Modifying <code>result</code> leaves <code>d1</code> unchanged. Using <code>result = d1</code> would make them point to the same object — any changes would affect <code>d1</code>.</p>
</div>

<div class="pb-card">
<div class="pb-card-num">02</div>
<h4>result.get(key, 0) handles both cases in one line</h4>
<p>If the key already exists in <code>result</code>, <code>.get()</code> returns its current value. If it's new (only in <code>d2</code>), it returns 0. Adding the <code>d2</code> value works correctly either way — no <code>if key in result</code> needed.</p>
</div>

<div class="pb-card">
<div class="pb-card-num">03</div>
<h4>d1.keys() | d2.keys() — set union of all keys</h4>
<p>Dict key views support set operations. <code>|</code> gives the union — all keys from both dicts without duplicates. Paired with <code>.get(k, 0)</code> for both dicts, this handles all three cases (d1-only, d2-only, both) uniformly.</p>
</div>
