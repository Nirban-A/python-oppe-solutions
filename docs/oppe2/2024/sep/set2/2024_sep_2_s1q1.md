???+ abstract "⚡ Quick Reference"
    **Function:** `is_both_divisible_by(a: int, b: int, c: int) -> bool`

    **Core idea:** check if `c` divides both `a` and `b` using the modulo operator.

    ```python
    def is_both_divisible_by(a: int, b: int, c: int) -> bool:
        return a % c == 0 and b % c == 0
    ```

    **Key rules:**
    - `x % c == 0` means `c` divides `x` exactly
    - Both conditions must hold - use `and`
    - `c` is guaranteed non-zero - no guard needed

---

## Problem Statement

!!! note "Problem"
    Write a function `is_both_divisible_by(a, b, c)` that returns `True` if both `a` and `b` are divisible by `c`.

## Solution approaches

=== "Pythonic (direct return)"

    ```python
    def is_both_divisible_by(a: int, b: int, c: int) -> bool:
        return a % c == 0 and b % c == 0
    ```

=== "Explanatory"

    ```python
    def is_both_divisible_by(a: int, b: int, c: int) -> bool:
        a_divisible = a % c == 0
        b_divisible = b % c == 0
        return a_divisible and b_divisible
    ```

=== "Using all()"

    ```python
    def is_both_divisible_by(a: int, b: int, c: int) -> bool:
        return all(n % c == 0 for n in (a, b))
    ```

    Scales naturally if the problem were extended to more numbers.

=== "Using lambda"

    ```python
    is_both_divisible_by = lambda a, b, c: a % c == 0 and b % c == 0
    ```

## Key takeaways

<div class="pb-cards">

<div class="pb-card">
<div class="pb-card-num">01</div>
<h4>x % c == 0 is the divisibility test</h4>
<p>The modulo operator gives the remainder. A remainder of 0 means <code>c</code> divides <code>x</code> exactly. This is the standard Python divisibility check.</p>
</div>

<div class="pb-card">
<div class="pb-card-num">02</div>
<h4>Short-circuit and - skip second check if first fails</h4>
<p><code>a % c == 0 and b % c == 0</code> - if <code>a</code> is not divisible by <code>c</code>, Python skips checking <code>b</code> entirely. The result is already <code>False</code>.</p>
</div>

<div class="pb-card">
<div class="pb-card-num">03</div>
<h4>all() for extensibility</h4>
<p><code>all(n % c == 0 for n in (a, b))</code> generalises naturally - add more numbers to the tuple without changing the logic. Good habit when checking the same condition across multiple values.</p>
</div>
