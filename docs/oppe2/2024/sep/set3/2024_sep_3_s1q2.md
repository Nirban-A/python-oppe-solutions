???+ abstract "⚡ Quick Reference"
    **Function:** `is_divisible_by_last_two_digits(num: int) -> bool`

    **Core idea:** extract the last two digits, guard against zero, check divisibility.

    ```python
    def is_divisible_by_last_two_digits(num: int):
        units = num % 10
        tens  = (num // 10) % 10
        if units == 0 or tens == 0:
            return False
        return num % units == 0 and num % tens == 0
    ```

    **Key rules:**
    - Last digit (units): `num % 10`
    - Second-to-last digit (tens): `(num // 10) % 10`
    - Return `False` if either digit is 0 (avoid division by zero)
    - Both `num % units == 0` and `num % tens == 0` must hold

!!! note "Problem"

## Solution approaches

=== "Pythonic"

    ```python
    def is_divisible_by_last_two_digits(num: int):
        units = num % 10
        tens  = (num // 10) % 10
        if units == 0 or tens == 0:
            return False
        return num % units == 0 and num % tens == 0
    ```

=== "Explanatory (named variables)"

    ```python
    def is_divisible_by_last_two_digits(num: int):
        last_digit        = num % 10
        second_last_digit = (num // 10) % 10

        # Guard: return False if either digit is zero
        if last_digit == 0 or second_last_digit == 0:
            return False

        divisible_by_last        = num % last_digit == 0
        divisible_by_second_last = num % second_last_digit == 0

        return divisible_by_last and divisible_by_second_last
    ```

=== "Using string conversion"

    ```python
    def is_divisible_by_last_two_digits(num: int):
        digits = [int(d) for d in str(num)[-2:]]   # last two digits
        if 0 in digits:
            return False
        return all(num % d == 0 for d in digits)
    ```

    `str(num)[-2:]` slices the last two characters of the string representation. Works cleanly for any number of digits.

=== "Using lambda"

    ```python
    def is_divisible_by_last_two_digits(num: int):
        u, t = num % 10, (num // 10) % 10
        return False if u == 0 or t == 0 else num % u == 0 and num % t == 0
    ```

## Key takeaways

<div class="pb-cards">

<div class="pb-card">
<div class="pb-card-num">01</div>
<h4>(num // 10) % 10 extracts the tens digit</h4>
<p>Dividing by 10 shifts the number right by one digit; taking the remainder mod 10 then isolates the new last digit. This pattern generalises — <code>(num // 100) % 10</code> gives the hundreds digit, and so on.</p>
</div>

<div class="pb-card">
<div class="pb-card-num">02</div>
<h4>Check for zero before dividing</h4>
<p>Dividing by zero raises a <code>ZeroDivisionError</code>. The zero guard must come before any <code>num % digit</code> call. Short-circuit <code>and</code> won't save you here — check zero explicitly first.</p>
</div>

<div class="pb-card">
<div class="pb-card-num">03</div>
<h4>Divisibility by the digit vs divisibility of the digit</h4>
<p>The question asks if <code>num</code> is divisible by its digits — i.e. <code>num % digit == 0</code>. A common mistake is checking if the digit divides into the last two digits instead of into <code>num</code> itself.</p>
</div>
