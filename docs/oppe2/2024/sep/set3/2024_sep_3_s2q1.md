???+ abstract "⚡ Quick Reference"
    **Function:** `get_words_after_the(sentence: str) -> list`

    **Core idea:** split the sentence into words, iterate with index, collect the word immediately after any word that equals `"the"` (case-insensitive).

    ```python
    def get_words_after_the(sentence: str) -> list:
        words = sentence.split()
        return [words[i + 1] for i in range(len(words) - 1)
                if words[i].lower() == "the"]
    ```

    **Key rules:**
    - Case-insensitive: `word.lower() == "the"`
    - Collect the word at `i + 1` whenever `words[i]` matches
    - Stop at `len(words) - 1` - the last word can't be followed by another
    - Preserve the original case of the word that follows

## Problem Statement

!!! note "Problem"

## Solution approaches

=== "Pythonic (list comprehension)"

    ```python
    def get_words_after_the(sentence: str) -> list:
        words = sentence.split()
        return [words[i + 1] for i in range(len(words) - 1)
                if words[i].lower() == "the"]
    ```

=== "Explanatory (loop)"

    ```python
    def get_words_after_the(sentence: str) -> list:
        words  = sentence.split()
        result = []
        for i in range(len(words) - 1):   # stop before last word
            if words[i].lower() == "the":
                result.append(words[i + 1])
        return result
    ```

=== "Using zip for adjacent pairs"

    ```python
    def get_words_after_the(sentence: str) -> list:
        words = sentence.split()
        return [b for a, b in zip(words, words[1:])
                if a.lower() == "the"]
    ```

    `zip(words, words[1:])` creates pairs of adjacent words `(words[i], words[i+1])`. Filter on the first element - elegant and no index arithmetic needed.

=== "Using lambda + filter"

    ```python
    def get_words_after_the(sentence: str) -> list:
        words = sentence.split()
        pairs = zip(words, words[1:])
        return list(map(lambda p: p[1],
                        filter(lambda p: p[0].lower() == "the", pairs)))
    ```

## Key takeaways

<div class="pb-cards">

<div class="pb-card">
<div class="pb-card-num">01</div>
<h4>range(len(words) - 1) avoids index out of bounds</h4>
<p>Checking <code>words[i+1]</code> requires <code>i+1</code> to be a valid index. Stopping at <code>len(words) - 1</code> ensures the last word (which has no successor) is never treated as a potential "the".</p>
</div>

<div class="pb-card">
<div class="pb-card-num">02</div>
<h4>zip(words, words[1:]) - clean adjacent pair pattern</h4>
<p>Pairing a list with its own slice shifted by one gives all consecutive pairs without explicit indexing. <code>zip(a, b)</code> stops at the shorter iterable - naturally skipping the last element.</p>
</div>

<div class="pb-card">
<div class="pb-card-num">03</div>
<h4>Preserve original case of the following word</h4>
<p>The search is case-insensitive (use <code>.lower()</code> on the potential "the"), but the collected word must keep its original casing. Never apply <code>.lower()</code> to <code>words[i+1]</code>.</p>
</div>
