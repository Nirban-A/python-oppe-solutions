???+ abstract "⚡ Quick Reference"
    **Function:** `is_even_two_digit_number(num: int) -> bool`

    **Core idea:** check two conditions — exactly two digits (ignoring sign) AND even.

    ```python
    def is_even_two_digit_number(num):
        return 10 <= abs(num) <= 99 and num % 2 == 0
    ```

    **Key rules:**
    - Two-digit: `10 ≤ abs(num) ≤ 99`
    - Even: `num % 2 == 0`
    - Negative numbers like `-36` count as two-digit (ignore the sign)
    - Both conditions must hold

---

## Problem Statement

!!! note "Problem"
    Write a function `is_even_two_digit_number(num)` that returns `True` if `num` is both a two-digit number (ignoring sign) and even.

## Solution approaches

=== "Pythonic (direct return)"

    ```python
    def is_even_two_digit_number(num):
        return 10 <= abs(num) <= 99 and num % 2 == 0
    ```

    Chained comparison `10 <= abs(num) <= 99` checks both bounds at once. Short-circuit `and` skips the even check if the number isn't two digits.

=== "Explanatory (named conditions)"

    ```python
    def is_even_two_digit_number(num):
        is_two_digit = 10 <= abs(num) <= 99
        is_even      = num % 2 == 0
        return is_two_digit and is_even
    ```

=== "Using len() on string"

    ```python
    def is_even_two_digit_number(num):
        is_two_digit = len(str(abs(num))) == 2
        is_even      = num % 2 == 0
        return is_two_digit and is_even
    ```

    Converts to string and checks length — useful when the digit-count approach feels more intuitive.

=== "Using lambda"

    ```python
    is_even_two_digit_number = lambda num: 10 <= abs(num) <= 99 and num % 2 == 0
    ```

## Key takeaways

<div class="pb-cards">

<div class="pb-card">
<div class="pb-card-num">01</div>
<h4>abs() handles the sign</h4>
<p><code>abs(num)</code> strips the negative sign before checking the digit range. <code>-36</code> has two digits (ignoring the sign) just like <code>36</code> — <code>abs(-36) = 36</code> which is in range.</p>
</div>

<div class="pb-card">
<div class="pb-card-num">02</div>
<h4>10 ≤ x ≤ 99 for two-digit numbers</h4>
<p>Two-digit numbers range from 10 to 99 inclusive. Python's chained comparison <code>10 <= x <= 99</code> is cleaner than <code>x >= 10 and x <= 99</code>.</p>
</div>

<div class="pb-card">
<div class="pb-card-num">03</div>
<h4>Check two-digit before even</h4>
<p>The two-digit check uses <code>abs(num)</code>; the even check uses <code>num</code>. Keep them separate — don't accidentally check <code>abs(num) % 2</code> for evenness, which would also make <code>-19</code> appear even.</p>
</div>
