???+ abstract "ÔÜí Quick Reference"
    **Function:** `is_dot_com_or_dot_in(domain: str) -> bool`

    **Core idea:** check if the string ends with `".com"` or `".in"` using `str.endswith()`.

    ```python
    def is_dot_com_or_dot_in(domain):
        return domain.endswith(".com") or domain.endswith(".in")
    ```

    **Key rules:**
    - Must end with `".com"` or `".in"` (the dot is part of the suffix)
    - `"invalidcom"` ÔåÆ `False` (no dot before `com`)
    - `str.endswith()` accepts a tuple ÔÇö can check both in one call

---

!!! note "Problem"

| `"mywebsitein"` | ÔØî | ÔØî (no dot) | `False` |

!!! insight "The dot is part of the suffix"
    `".com"` includes the dot ÔÇö so `"invalidcom"` correctly returns `False` since it doesn't have a `.` before `com`. Always include the dot in the suffix string.

## Solution approaches

=== "Pythonic (endswith with tuple)"

    ```python
    def is_dot_com_or_dot_in(domain):
        return domain.endswith((".com", ".in"))
    ```

    `str.endswith()` accepts a **tuple of suffixes** and returns `True` if the string ends with any of them. The cleanest single-expression solution.

=== "Explanatory (two separate checks)"

    ```python
    def is_dot_com_or_dot_in(domain):
        return domain.endswith(".com") or domain.endswith(".in")
    ```

=== "Using string slicing"

    ```python
    def is_dot_com_or_dot_in(domain):
        return domain[-4:] == ".com" or domain[-3:] == ".in"
    ```

    Slice the last 4 characters for `.com` and last 3 for `.in`. Less readable than `endswith()` but shows the underlying idea.

=== "Using lambda"

    ```python
    is_dot_com_or_dot_in = lambda domain: domain.endswith((".com", ".in"))
    ```

## Key takeaways

<div class="pb-cards">

<div class="pb-card">
<div class="pb-card-num">01</div>
<h4>str.endswith() accepts a tuple</h4>
<p><code>domain.endswith((".com", ".in"))</code> checks both suffixes in a single call. This is more Pythonic than using <code>or</code> with two separate <code>endswith()</code> calls.</p>
</div>

<div class="pb-card">
<div class="pb-card-num">02</div>
<h4>Always include the dot in the suffix</h4>
<p>Searching for <code>".com"</code> (with dot) correctly rejects <code>"invalidcom"</code>. Searching for <code>"com"</code> (without dot) would incorrectly accept it.</p>
</div>

<div class="pb-card">
<div class="pb-card-num">03</div>
<h4>endswith() vs in</h4>
<p><code>".com" in domain</code> would also match <code>"my.company.org"</code> since <code>.com</code> appears in the middle. Always use <code>endswith()</code> for suffix checks ÔÇö not <code>in</code>.</p>
</div>
