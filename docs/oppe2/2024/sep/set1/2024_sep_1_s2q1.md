???+ abstract "⚡ Quick Reference"
    **Function:** `count_odd_three_digit_nums(nums: list) -> int`

    **Core idea:** filter out `None`, then count elements that are three-digit (using `abs`) and odd.

    ```python
    def count_odd_three_digit_nums(nums):
        return sum(
            1 for n in nums
            if n is not None
            and 100 <= abs(n) <= 999
            and n % 2 != 0
        )
    ```

    **Key rules:**
    - Skip `None` values — check `n is not None` first
    - Three-digit: `100 ≤ abs(n) ≤ 999` (ignores negative sign)
    - Odd: `n % 2 != 0`
    - All three conditions must hold

## Solution approaches

=== "Pythonic (generator expression)"

    ```python
    def count_odd_three_digit_nums(nums):
        return sum(
            1 for n in nums
            if n is not None
            and 100 <= abs(n) <= 999
            and n % 2 != 0
        )
    ```

=== "Explanatory (loop with counter)"

    ```python
    def count_odd_three_digit_nums(nums):
        count = 0
        for n in nums:
            if n is None:
                continue
            if 100 <= abs(n) <= 999 and n % 2 != 0:
                count += 1
        return count
    ```

=== "Using filter + lambda"

    ```python
    def count_odd_three_digit_nums(nums):
        valid = filter(
            lambda n: n is not None and 100 <= abs(n) <= 999 and n % 2 != 0,
            nums
        )
        return len(list(valid))
    ```

=== "Using list comprehension + len"

    ```python
    def count_odd_three_digit_nums(nums):
        return len([
            n for n in nums
            if n is not None
            and 100 <= abs(n) <= 999
            and n % 2 != 0
        ])
    ```

## Key takeaways

<div class="pb-cards">

<div class="pb-card">
<div class="pb-card-num">01</div>
<h4>Check None first with "is not None"</h4>
<p>Always check <code>n is not None</code> before calling <code>abs(n)</code> or <code>n % 2</code>. Calling <code>abs(None)</code> raises a <code>TypeError</code>. Python's short-circuit <code>and</code> ensures the later conditions only run when <code>n</code> is not <code>None</code>.</p>
</div>

<div class="pb-card">
<div class="pb-card-num">02</div>
<h4>abs() handles negative three-digit numbers</h4>
<p><code>abs(-203) = 203</code> which is in range <code>[100, 999]</code>. Without <code>abs()</code>, negative numbers would always fail the range check since <code>-203 &lt; 100</code>.</p>
</div>

<div class="pb-card">
<div class="pb-card-num">03</div>
<h4>n % 2 != 0 for odd — use original n, not abs(n)</h4>
<p>In Python, <code>-203 % 2 == 1</code> (odd) and <code>-204 % 2 == 0</code> (even), so the odd check works correctly on negative numbers directly. Using <code>abs(n) % 2</code> would also work but is redundant.</p>
</div>
