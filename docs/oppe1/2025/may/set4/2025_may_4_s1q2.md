???+ abstract "⚡ Quick Reference"
    **Function:** `extract_email_username(email: str) -> str`

    **Core idea:** split on `@` and return the first part.

    ```python
    def extract_email_username(email: str) -> str:
        return email.split("@")[0]
    ```

    **Key rules:**
    - Exactly one `@` in the input -guaranteed
    - Everything before `@` is the username
    - `split("@")[0]` is the simplest and most direct approach

## Problem Statement

!!! note "Problem"

## Understanding the problem

An email address has exactly one `@` that separates the username from the domain. Everything to the left of `@` is what we need:

```
"ananya.sharma@iitd.ac.in"
               ↑
              "@" is the separator

Left:  "ananya.sharma"  ← return this
Right: "iitd.ac.in"    ← ignore
```

---

## Solution approaches

=== "Pythonic (split)"

    ```python
    def extract_email_username(email: str) -> str:
        return email.split("@")[0]
    ```

    `split("@")` produces `["ananya.sharma", "iitd.ac.in"]`. Index `[0]` gives the username. Clean and direct.

=== "Using str.partition()"

    ```python
    def extract_email_username(email: str) -> str:
        username, _, _ = email.partition("@")
        return username
    ```

    `str.partition(sep)` splits into exactly three parts: `(before, sep, after)`. Slightly more explicit than `split` -makes it clear you're splitting on the first occurrence of `@` and taking the left part.

=== "Using index slicing"

    ```python
    def extract_email_username(email: str) -> str:
        at_pos = email.index("@")
        return email[:at_pos]
    ```

    `str.index("@")` finds the position of `@`. Slice up to that position. More verbose but shows the underlying mechanism clearly.

=== "Using lambda"

    ```python
    extract_email_username = lambda email: email.split("@")[0]
    ```

    The entire function as a lambda -useful when passing as an argument to `map()`.

## Key takeaways

<div class="pb-cards">

<div class="pb-card">
<div class="pb-card-num">01</div>
<h4>split(sep)[0] for "everything before"</h4>
<p><code>s.split(sep)[0]</code> is the standard pattern for extracting everything before a known separator. Works for <code>@</code>, <code>:</code>, <code>.</code>, or any delimiter.</p>
</div>

<div class="pb-card">
<div class="pb-card-num">02</div>
<h4>str.partition() vs str.split()</h4>
<p><code>partition()</code> always returns exactly three parts and never creates extra splits. <code>split()</code> splits on every occurrence. For a guaranteed single separator like <code>@</code>, both work equally well.</p>
</div>

<div class="pb-card">
<div class="pb-card-num">03</div>
<h4>str.index() vs str.find()</h4>
<p><code>index()</code> raises a <code>ValueError</code> if the separator isn't found. <code>find()</code> returns <code>-1</code>. Since <code>@</code> is guaranteed present, <code>index()</code> is the safer choice -it fails loudly on bad input.</p>
</div>
