???+ abstract "⚡ Quick Reference"
    **Type:** Full I/O -positional arrow drawing

    **Core idea:** track current position; for each delta draw the arrow with leading spaces based on its start column.

    ```python
    deltas = list(map(int, input().split()))
    pos = 0
    for d in deltas:
        if d > 0:
            print(" " * pos + "=" * d + ">")
        elif d < 0:
            v = -d
            print(" " * (pos - v) + "<" + "=" * (v - 1))
        else:
            print(" " * pos + "=")
        pos += d
    ```

    **Key rules:**
    - Right (`d > 0`): arrow starts at `pos`, `"=" * d + ">"`, leading spaces = `pos`
    - Left (`d < 0`): arrow ends at `pos`, `"<" + "=" * (|d|-1)`, leading spaces = `pos - |d|`
    - Zero: `"="` at `pos`, leading spaces = `pos`
    - Update `pos += d` after printing each arrow

## Problem Statement

!!! note "Problem (I/O type)"
    Read space-separated movement deltas. For each delta, draw an arrow aligned to its position. Right arrows start at the current position; left arrows end at it.

**Example:**

## Understanding the arrow rules

**Right arrow** (`d > 0`):
- Starts at column `pos`
- Length `d` equals signs followed by `>`
- Content: `"=" * d + ">"`
- Leading spaces: `pos`

**Left arrow** (`d < 0`, length `v = -d`):
- **Ends** at column `pos` (current position before moving)
- Starts at column `pos - v`
- Content: `"<" + "=" * (v - 1)` -total length `v`
- Leading spaces: `pos - v`

**Zero** (`d == 0`):
- Single `=` at column `pos`
- Leading spaces: `pos`

!!! insight "Left arrow alignment"
    The left arrow *ends* at the current position, so it starts at `pos - v`. The `<` is at the leftmost end, followed by `v-1` equals signs. For `d=-2` at `pos=5`: starts at column 3, content `<==`, leading spaces `5-2=3`.

## Solution approaches

=== "Pythonic"

    ```python
    deltas = list(map(int, input().split()))
    pos = 0
    for d in deltas:
        if d > 0:
            print(" " * pos + "=" * d + ">")
        elif d < 0:
            v = -d
            print(" " * (pos - v) + "<" + "=" * (v - 1))
        else:
            print(" " * pos + "=")
        pos += d
    ```

=== "Explanatory (named variables)"

    ```python
    deltas = list(map(int, input().split()))
    pos = 0
    for d in deltas:
        if d > 0:
            padding = " " * pos
            arrow   = "=" * d + ">"
        elif d < 0:
            v       = abs(d)
            padding = " " * (pos - v)
            arrow   = "<" + "=" * (v - 1)
        else:
            padding = " " * pos
            arrow   = "="
        print(padding + arrow)
        pos += d
    ```

    Every component named -easy to trace and debug.

=== "Using a helper function"

    ```python
    def make_arrow(d, pos):
        if d > 0:
            return " " * pos + "=" * d + ">"
        elif d < 0:
            v = -d
            return " " * (pos - v) + "<" + "=" * (v - 1)
        return " " * pos + "="

    deltas = list(map(int, input().split()))
    pos = 0
    for d in deltas:
        print(make_arrow(d, pos))
        pos += d
    ```

=== "Using lambda"

    ```python
    make_arrow = lambda d, pos: (
        " " * pos + "=" * d + ">"          if d > 0 else
        " " * (pos + d) + "<" + "=" * (-d - 1) if d < 0 else
        " " * pos + "="
    )

    deltas = list(map(int, input().split()))
    pos = 0
    for d in deltas:
        print(make_arrow(d, pos))
        pos += d
    ```

    Note: for negative `d`, `pos + d` = `pos - |d|` -the start column of the left arrow.

## Key takeaways

<div class="pb-cards">

<div class="pb-card">
<div class="pb-card-num">01</div>
<h4>Right arrow starts at pos, left arrow ends at pos</h4>
<p>The current position is always the <em>anchor</em> -right arrows grow rightward from it, left arrows grow leftward ending at it. Getting this direction right is the key insight.</p>
</div>

<div class="pb-card">
<div class="pb-card-num">02</div>
<h4>Left arrow leading spaces = pos - |d|</h4>
<p>The left arrow starts <code>|d|</code> columns to the left of the current position. Leading spaces = <code>pos - |d|</code>. Always guaranteed non-negative since the problem states position is always ≥ 0.</p>
</div>

<div class="pb-card">
<div class="pb-card-num">03</div>
<h4>Update pos after printing</h4>
<p>Print with the current <code>pos</code>, then update with <code>pos += d</code>. Updating before printing would shift every arrow by one step -a common off-by-one mistake in positional problems.</p>
</div>
