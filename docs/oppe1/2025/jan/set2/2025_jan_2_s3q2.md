???+ abstract "⚡ Quick Reference"
    **Type:** Full I/O — pattern printing

    **Core idea:** top and bottom rows are full stars; diagonal rows have a star at position `n - i` (0-indexed) with leading spaces only — no trailing spaces.

    ```python
    n = int(input())
    for i in range(1, n + 1):
        if i == 1 or i == n:
            print("*" * n)
        else:
            print(" " * (n - i) + "*")
    ```

    **Key rules:**
    - Row 1 and row `n`: full row of `n` stars
    - Rows 2 to `n-1`: `n - i` leading spaces then `*` (no trailing spaces)
    - `i` is 1-indexed

---

## Problem Statement

!!! note "Problem (I/O type)"
    Given `n`, print a Z-shaped pattern:
    - Top row: `n` stars
    - Diagonal: star moves from right to left as rows increase
    - Bottom row: `n` stars
    - No trailing spaces after the last `*` on any line

## Deriving the pattern

For `n = 5`, mark where the `*` appears on each row (1-indexed):

| Row `i` | Type | Leading spaces | Star position |
|---------|------|---------------|--------------|
| 1 | Top border | — | All `n` stars |
| 2 | Diagonal | 3 | `n - i = 3` |
| 3 | Diagonal | 2 | `n - i = 2` |
| 4 | Diagonal | 1 | `n - i = 1` |
| 5 | Bottom border | — | All `n` stars |

The diagonal star starts at the **second-to-last** column (`n-2` in 0-indexed, which is `n-i` spaces for `i=2`) and moves one step left each row, reaching column 0 at row `n-1`.

Formula for diagonal rows: `" " * (n - i) + "*"`

!!! insight "No trailing spaces"
    The problem explicitly says no space after the last `*`. Since the diagonal rows have their star at a position left of the rightmost column, there would be trailing spaces if you tried to pad to width `n`. The formula `" " * (n - i) + "*"` naturally produces no trailing spaces.

## Solution approaches

=== "Pythonic"

    ```python
    n = int(input())
    for i in range(1, n + 1):
        if i == 1 or i == n:
            print("*" * n)
        else:
            print(" " * (n - i) + "*")
    ```

=== "Explanatory"

    ```python
    n = int(input())
    for i in range(1, n + 1):
        if i == 1 or i == n:
            row = "*" * n                   # full row of stars
        else:
            leading_spaces = " " * (n - i)  # decreases each row
            row = leading_spaces + "*"       # no trailing spaces
        print(row)
    ```

=== "Using f-string"

    ```python
    n = int(input())
    for i in range(1, n + 1):
        if i == 1 or i == n:
            print("*" * n)
        else:
            print(f"{'*':>{n - i + 1}}")
    ```

    `f"{'*':>{n - i + 1}}"` right-aligns `*` in a field of width `n - i + 1`. This produces `n - i` leading spaces followed by `*` — no trailing spaces.

=== "Using lambda"

    ```python
    n = int(input())
    row = lambda i: "*" * n if i == 1 or i == n else " " * (n - i) + "*"
    for i in range(1, n + 1):
        print(row(i))
    ```

    The `lambda` encapsulates the row-building logic — pass the row number, get the row string. Clean separation of the pattern logic from the loop.

---

## Common mistake — trailing spaces

A common mistake is printing each row padded to width `n`:

```python
# ❌ Wrong — adds trailing spaces to diagonal rows
print(f"{'*':>{n - i + 1}:<{n}}")   # pads right side too
```

The problem says no trailing spaces. Always end diagonal rows right after the `*`.

## Key takeaways

<div class="pb-cards">

<div class="pb-card">
<div class="pb-card-num">01</div>
<h4>Border rows vs diagonal rows</h4>
<p>Pattern problems almost always have special cases for the first and last row. Handle them with <code>if i == 1 or i == n</code> and apply the general formula for everything in between.</p>
</div>

<div class="pb-card">
<div class="pb-card-num">02</div>
<h4>No trailing spaces — end at the star</h4>
<p><code>" " * (n - i) + "*"</code> stops right after the star. Never pad to the full row width when trailing spaces are forbidden.</p>
</div>

<div class="pb-card">
<div class="pb-card-num">03</div>
<h4>Diagonal moves left as i increases</h4>
<p>For Z: star starts at the right and moves left. Leading spaces = <code>n - i</code> — decreases as <code>i</code> increases. Contrast with triangles where stars grow; here only one star moves.</p>
</div>
