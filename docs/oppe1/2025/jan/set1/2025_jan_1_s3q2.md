???+ abstract "ÔÜí Quick Reference"
    **Type:** Full I/O ÔÇö pattern printing

    **Core idea:** row `i` (1-indexed from top) has `n-i` spaces between `|` and `/`, then `2*(i-1)` inner spaces, then `\`, then `n-i` spaces before the closing `|`.

    ```python
    n = int(input())
    for i in range(1, n + 1):
        pad = " " * (n - i)
        inner = " " * (2 * (i - 1))
        print(f"|{pad}/{inner}\\{pad}|")
    ```

    **Key rules:**
    - Outer `|` on both sides of every row
    - `/` and `\` move diagonally inward as rows increase
    - Padding outside `/` and `\`: `n - i` spaces (decreases row by row)
    - Inner spaces between `/` and `\`: `2 * (i - 1)` (increases row by row)
    - Escape `\` in Python strings as `\\`

## Deriving the pattern

Look at each row for `n = 3`:

```
Row 1:  |  /\  |     ÔåÆ  | + 2 spaces + / + 0 inner spaces + \ + 2 spaces + |
Row 2:  | /  \ |     ÔåÆ  | + 1 space  + / + 2 inner spaces + \ + 1 space  + |
Row 3:  |/    \|     ÔåÆ  | + 0 spaces + / + 4 inner spaces + \ + 0 spaces + |
```

Two formulas for row `i` (1-indexed from top):

| Part | Formula | Row 1 | Row 2 | Row 3 |
|------|---------|-------|-------|-------|
| Outer padding (each side) | `n - i` | 2 | 1 | 0 |
| Inner spaces (between `/` and `\`) | `2 * (i - 1)` | 0 | 2 | 4 |

Row structure: `|` + `(n-i)` spaces + `/` + `2*(i-1)` spaces + `\` + `(n-i)` spaces + `|`

!!! insight "Why 2*(i-1) inner spaces?"
    The `/` and `\` both move one step outward each row ÔÇö `/` moves one step left, `\` moves one step right. Together they create 2 extra inner spaces per row. At row 1 there are 0 inner spaces; at row 2 there are 2; at row 3 there are 4 ÔÇö each time increasing by 2.

## The backslash problem

In Python, `\` is an escape character in strings. To print a literal backslash you must write `\\`:

```python
print("\\")   # prints:  \
print("/")    # prints:  /  (no escaping needed)
```

In an f-string:
```python
pad = "  "
print(f"|{pad}/\\{pad}|")   # prints:  |  /\  |
```

## Solution approaches

=== "Pythonic (f-string)"

    ```python
    n = int(input())
    for i in range(1, n + 1):
        pad   = " " * (n - i)
        inner = " " * (2 * (i - 1))
        print(f"|{pad}/{inner}\\{pad}|")
    ```

    Clean and readable. Two variables ÔÇö `pad` for outer spacing, `inner` for the space between `/` and `\`.

=== "Explanatory (string concatenation)"

    ```python
    n = int(input())
    for i in range(1, n + 1):
        outer_spaces = " " * (n - i)
        inner_spaces = " " * (2 * (i - 1))
        row = "|" + outer_spaces + "/" + inner_spaces + "\\" + outer_spaces + "|"
        print(row)
    ```

    Build each part as a named string and concatenate. Every piece of the row is visible.

=== "One-liner"

    ```python
    n = int(input())
    for i in range(1, n + 1):
        print("|" + " "*(n-i) + "/" + " "*(2*(i-1)) + "\\" + " "*(n-i) + "|")
    ```

    Fully inline ÔÇö no intermediate variables. Compact but harder to read at a glance.

## Verifying with n = 1

`i = 1`: `n - i = 0`, `2*(i-1) = 0`

Row: `|` + `` + `/` + `` + `\` + `` + `|` = `|/\|` Ô£ô

## Key takeaways

<div class="pb-card">
<div class="pb-card-num">01</div>
<h4>Escape backslash as \\</h4>
<p><code>\</code> is an escape character in Python strings. Always write <code>\\</code> to get a literal backslash in output. In f-strings, <code>f"{\\"}</code> is not valid ÔÇö use a variable: <code>bs = "\\"; f"{bs}"</code> or write <code>"\\"</code> directly outside the braces.</p>
</div>

<div class="pb-card">
<div class="pb-card-num">02</div>
<h4>Symmetric padding on both sides</h4>
<p>The outer padding is the same on the left and right of each row (<code>n - i</code>). Store it in one variable and reuse it for both sides ÔÇö less chance of a typo.</p>
</div>

<div class="pb-card">
<div class="pb-card-num">03</div>
<h4>Inner spaces grow by 2 per row</h4>
<p>Each step down, <code>/</code> moves one left and <code>\</code> moves one right ÔÇö together adding 2 inner spaces. The formula <code>2*(i-1)</code> captures this: 0 at row 1, 2 at row 2, 4 at row 3, ÔÇª</p>
</div>
