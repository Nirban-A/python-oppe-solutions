???+ abstract "⚡ Quick Reference"
    **Function:** `sum_squares_abs_diff_squares(a: int, b: int) -> tuple`

    **Core idea:** compute `a²` and `b²`, return both their sum and their absolute difference.

    ```python
    def sum_squares_abs_diff_squares(a: int, b: int) -> tuple:
        return (a**2 + b**2, abs(a**2 - b**2))
    ```

    **Key rules:**
    - First element: `a² + b²`
    - Second element: `|a² - b²|`
    - `abs()` handles the case where `b² > a²`

---

## Problem Statement

!!! note "Problem"
    Write a function `sum_squares_abs_diff_squares(a, b)` that returns a tuple of:
    1. The sum of the squares: `a² + b²`
    2. The absolute difference of the squares: `|a² - b²|`

**Example:**

<div class="pb-io">
<div class="pb-io-block input">
<span class="pb-io-label">Input</span>
<pre><code>a=3, b=4</code></pre>
</div>
<div class="pb-io-block output">
<span class="pb-io-label">Output</span>
<pre><code>(25, 7)</code></pre>
</div>
</div>

Verification: `3²=9`, `4²=16`, sum=`9+16=25`, abs diff=`|9-16|=7` 

---

## Solution approaches

=== "Pythonic (one-liner)"

    ```python
    def sum_squares_abs_diff_squares(a: int, b: int) -> tuple:
        return (a**2 + b**2, abs(a**2 - b**2))
    ```

=== "Explanatory (named variables)"

    ```python
    def sum_squares_abs_diff_squares(a: int, b: int) -> tuple:
        sq_a = a ** 2
        sq_b = b ** 2
        sum_sq  = sq_a + sq_b
        diff_sq = abs(sq_a - sq_b)
        return (sum_sq, diff_sq)
    ```

=== "Using pow()"

    ```python
    def sum_squares_abs_diff_squares(a: int, b: int) -> tuple:
        sq_a, sq_b = pow(a, 2), pow(b, 2)
        return (sq_a + sq_b, abs(sq_a - sq_b))
    ```

    `pow(a, 2)` is equivalent to `a**2`. Compute each square once and reuse.

=== "Using lambda"

    ```python
    sum_squares_abs_diff_squares = lambda a, b: (a**2 + b**2, abs(a**2 - b**2))
    ```

## Key takeaways

<div class="pb-cards">

<div class="pb-card">
<div class="pb-card-num">01</div>
<h4>abs() for absolute difference</h4>
<p><code>abs(a² - b²)</code> handles both cases: when <code>a² > b²</code> and when <code>b² > a²</code>. Never write <code>if a**2 > b**2: ... else: ...</code> when <code>abs()</code> does it in one call.</p>
</div>

<div class="pb-card">
<div class="pb-card-num">02</div>
<h4>Compute squares once, reuse</h4>
<p>If using the explanatory approach, store <code>sq_a</code> and <code>sq_b</code> - avoids computing <code>a**2</code> twice. For two values it's minor, but it's a good habit.</p>
</div>

<div class="pb-card">
<div class="pb-card-num">03</div>
<h4>Return a tuple, not a list</h4>
<p>The problem specifies a <code>tuple</code> return type. <code>(x, y)</code> creates a tuple; <code>[x, y]</code> creates a list. They're different types - check the expected return carefully.</p>
</div>
