???+ abstract "⚡ Quick Reference"
    **Function:** `is_odd_length_palindrome(s: str) -> bool`

    **Core idea:** two conditions must both be true - odd length AND palindrome.

    ```python
    def is_odd_length_palindrome(s: str) -> bool:
        return len(s) % 2 == 1 and s == s[::-1]
    ```

    **Key rules:**
    - Odd length → `len(s) % 2 == 1`
    - Palindrome → `s == s[::-1]`
    - Both must hold → use `and`

---

## Problem Statement

!!! note "Problem"
    Write a function `is_odd_length_palindrome(s)` that returns `True` if `s` is both a palindrome **and** has an odd number of characters, `False` otherwise.

## Solution approaches

=== "Pythonic (direct return)"

    ```python
    def is_odd_length_palindrome(s: str) -> bool:
        return len(s) % 2 == 1 and s == s[::-1]
    ```

=== "Explanatory (if-else)"

    ```python
    def is_odd_length_palindrome(s: str) -> bool:
        is_odd = len(s) % 2 == 1
        is_palindrome = s == s[::-1]
        return is_odd and is_palindrome
    ```

=== "Short-circuit optimised"

    ```python
    def is_odd_length_palindrome(s: str) -> bool:
        if len(s) % 2 == 0:      # even length → fail immediately
            return False
        return s == s[::-1]
    ```

    Check the cheaper condition (length modulo) first. If it fails, skip the string reversal entirely.

=== "Using lambda"

    ```python
    is_odd_length_palindrome = lambda s: len(s) % 2 == 1 and s == s[::-1]
    ```

## Key takeaways

<div class="pb-cards">

<div class="pb-card">
<div class="pb-card-num">01</div>
<h4>Two independent conditions - use and</h4>
<p>Both must be true simultaneously. <code>and</code> short-circuits - if length is even, the palindrome check is never performed.</p>
</div>

<div class="pb-card">
<div class="pb-card-num">02</div>
<h4>s == s[::-1] for palindrome</h4>
<p>The one-liner palindrome check. Works for any sequence. Odd-length palindromes always have a unique middle character - but you don't need to find it explicitly.</p>
</div>

<div class="pb-card">
<div class="pb-card-num">03</div>
<h4>Check the cheap condition first</h4>
<p>Length check is O(1); reversal is O(n). Putting the length check first means even-length strings never trigger the reversal - a small but clean optimisation.</p>
</div>
