???+ abstract "⚡ Quick Reference"
    **Function:** `sum_of_squares_of_even(nums: list) -> int`

    **Core idea:** filter even numbers, square each one, sum them all.

    ```python
    def sum_of_squares_of_even(nums: list) -> int:
        return sum(x**2 for x in nums if x % 2 == 0)
    ```

    **Key rules:**
    - Even → `x % 2 == 0`
    - Square → `x**2`
    - Sum everything matching the condition

## Problem Statement

!!! note "Problem"

## Solution approaches

=== "Pythonic (generator + sum)"

    ```python
    def sum_of_squares_of_even(nums: list) -> int:
        return sum(x**2 for x in nums if x % 2 == 0)
    ```

=== "Explanatory (loop)"

    ```python
    def sum_of_squares_of_even(nums: list) -> int:
        total = 0
        for x in nums:
            if x % 2 == 0:
                total += x ** 2
        return total
    ```

=== "Using filter + map"

    ```python
    def sum_of_squares_of_even(nums: list) -> int:
        evens = filter(lambda x: x % 2 == 0, nums)
        return sum(map(lambda x: x**2, evens))
    ```

    `filter` selects even numbers; `map` squares them; `sum` totals them. Clean functional pipeline.

=== "Using list comprehension"

    ```python
    def sum_of_squares_of_even(nums: list) -> int:
        return sum([x**2 for x in nums if x % 2 == 0])
    ```

    Explicit list comprehension - same result as the generator but materialises the list in memory first. For large lists, the generator approach is more memory-efficient.

=== "Using lambda + reduce"

    ```python
    from functools import reduce

    def sum_of_squares_of_even(nums: list) -> int:
        return reduce(lambda acc, x: acc + x**2 if x % 2 == 0 else acc, nums, 0)
    ```

    `reduce` accumulates the sum directly - adds `x²` if `x` is even, leaves the accumulator unchanged otherwise.

## Key takeaways

<div class="pb-cards">

<div class="pb-card">
<div class="pb-card-num">01</div>
<h4>sum(x**2 for x in lst if cond)</h4>
<p>The standard pattern for "filter then transform then sum". The condition goes inside the generator - no separate filter step needed.</p>
</div>

<div class="pb-card">
<div class="pb-card-num">02</div>
<h4>Generator vs list comprehension</h4>
<p><code>sum(x**2 for x in ...)</code> uses a generator - values are computed one at a time without building a full list. <code>sum([x**2 for x in ...])</code> builds the list first. For large inputs, generators use less memory.</p>
</div>

<div class="pb-card">
<div class="pb-card-num">03</div>
<h4>filter + map + sum pipeline</h4>
<p>Three separate operations chained together. Each function does one thing - filter selects, map transforms, sum aggregates. This separation of concerns makes each step independently readable.</p>
</div>
