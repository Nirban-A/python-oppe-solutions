???+ abstract "⚡ Quick Reference"
    **Function:** `div_by_exactly_one(num: int, a: int, b: int) -> bool`

    **Core idea:** XOR - true when exactly one of two conditions is true.

    ```python
    def div_by_exactly_one(num: int, a: int, b: int) -> bool:
        return (num % a == 0) != (num % b == 0)
    ```

    **Key rules:**
    - Divisible by `a` but not `b` → `True`
    - Divisible by `b` but not `a` → `True`
    - Divisible by both → `False`
    - Divisible by neither → `False`
    - `!=` on two booleans is XOR

---

## Understanding the problem

"Exactly one" means the XOR of two boolean conditions:

| div by `a`? | div by `b`? | Exactly one? |
|------------|------------|-------------|
| ✅ | ❌ | ✅ `True` |
| ❌ | ✅ | ✅ `True` |
| ✅ | ✅ | ❌ `False` (both) |
| ❌ | ❌ | ❌ `False` (neither) |

!!! insight "!= as XOR for booleans"
    For two boolean values, `p != q` is exactly XOR - `True` when exactly one is `True`. So `(num % a == 0) != (num % b == 0)` is the most compact expression for "exactly one".

## Solution approaches

=== "Pythonic (XOR with !=)"

    ```python
    def div_by_exactly_one(num: int, a: int, b: int) -> bool:
        return (num % a == 0) != (num % b == 0)
    ```

=== "Explanatory (if-elif)"

    ```python
    def div_by_exactly_one(num: int, a: int, b: int) -> bool:
        div_a = num % a == 0
        div_b = num % b == 0
        if div_a and div_b:
            return False
        elif div_a or div_b:
            return True
        else:
            return False
    ```

=== "Using ^ operator"

    ```python
    def div_by_exactly_one(num: int, a: int, b: int) -> bool:
        return (num % a == 0) ^ (num % b == 0)
    ```

    Python's `^` is bitwise XOR. For booleans, it's equivalent to `!=`.

=== "Using lambda"

    ```python
    div_by_exactly_one = lambda num, a, b: (num % a == 0) != (num % b == 0)
    ```

## Key takeaways

<div class="pb-cards">

<div class="pb-card">
<div class="pb-card-num">01</div>
<h4>!= is XOR for booleans</h4>
<p><code>p != q</code> returns <code>True</code> when exactly one of <code>p</code> or <code>q</code> is <code>True</code>. The cleanest way to express "exactly one of two conditions".</p>
</div>

<div class="pb-card">
<div class="pb-card-num">02</div>
<h4>^ is explicit XOR</h4>
<p><code>p ^ q</code> works identically to <code>p != q</code> for booleans. More readable to someone familiar with logic/CS notation.</p>
</div>

<div class="pb-card">
<div class="pb-card-num">03</div>
<h4>"Exactly one" vs "at least one"</h4>
<p>"Exactly one" → XOR. "At least one" → OR. "Both" → AND. "Neither" → NOR. Recognise which the problem asks for before writing the condition.</p>
</div>
