???+ abstract "⚡ Quick Reference"
    **Type:** Pattern printing - I/O (write complete code)

    **Core idea:** Row `i` (1-indexed) has `2i - 1` stars, centred in a field of width `2n - 1`.

    ```python
    n = int(input())
    width = 2 * n - 1
    for i in range(1, n + 1):
        stars = "*" * (2 * i - 1)
        print(stars.center(width))
    ```

    **Key rules:**
    - Row 1 → 1 star, Row 2 → 3 stars, … Row n → `2n-1` stars
    - Total width = `2n - 1` (widest row, bottom)
    - `str.center(width)` pads equally on both sides
    - For odd widths centering is exact; `str.center()` handles this automatically

## Problem Statement

!!! note "Problem (I/O type)"
    Given an integer `n`, print a triangle of `*` where:

    - Row `i` contains `2i - 1` stars (so row 1 has 1, row 2 has 3, …, row `n` has `2n-1`)
    - Each row is **centred** relative to the widest row

## Deriving the formula

Think about what each row needs:

| Row `i` | Stars needed | Formula |
|---------|-------------|---------|
| 1 | 1 | `2(1) - 1 = 1` |
| 2 | 3 | `2(2) - 1 = 3` |
| 3 | 5 | `2(3) - 1 = 5` |
| 4 | 7 | `2(4) - 1 = 7` |

Stars grow by 2 each row. Row `i` has `2i - 1` stars.

The **total width** is determined by the bottom row: `2n - 1` characters.

To centre row `i` inside this width, we need:
```
leading_spaces = (width - stars_in_row) // 2
               = ((2n-1) - (2i-1)) // 2
               = (n - i) spaces
```

So row 1 has `n-1` leading spaces, row 2 has `n-2`, …, row `n` has 0.

!!! insight "Key insight"
    You can compute spaces manually with `(n - i) * " "`, or just use `str.center(width)` and let Python do the arithmetic. Both approaches produce identical output.

## Step-by-step walkthrough

### Step 1 - Read n and compute the total width

```python
n = int(input())
width = 2 * n - 1   # widest row = bottom row
```

### Step 2 - Loop from row 1 to row n

```python
for i in range(1, n + 1):
```

### Step 3 - Build the star string for row i

```python
stars = "*" * (2 * i - 1)
```

### Step 4 - Centre it and print

```python
print(stars.center(width))
```

`"*".center(7)` → `"   *   "` (width 7, centred).

## Solution approaches

=== "Pythonic (str.center)"

    ```python
    n = int(input())
    width = 2 * n - 1
    for i in range(1, n + 1):
        print(("*" * (2 * i - 1)).center(width))
    ```

    Clean, no manual space arithmetic. `str.center()` handles everything.

=== "Manual spacing"

    ```python
    n = int(input())
    for i in range(1, n + 1):
        spaces = " " * (n - i)
        stars  = "*" * (2 * i - 1)
        print(spaces + stars)
    ```

    Explicit calculation: `n - i` leading spaces. No trailing spaces (which is fine since `print` adds a newline and trailing spaces are invisible).

=== "f-string with format spec"

    ```python
    n = int(input())
    width = 2 * n - 1
    for i in range(1, n + 1):
        stars = "*" * (2 * i - 1)
        print(f"{stars:^{width}}")
    ```

    `^` in a format spec means "centre-align". `{stars:^{width}}` centres `stars` in a field of `width` characters. This is equivalent to `stars.center(width)`.

## Key takeaways

<div class="pb-card">
<div class="pb-card-num">01</div>
<h4>str.center(width)</h4>
<p>Centres a string in a field of given width, padding with spaces on both sides. The cleanest way to centre-align pattern rows.</p>
</div>

<div class="pb-card">
<div class="pb-card-num">02</div>
<h4>f-string align: <code>^</code></h4>
<p>In an f-string format spec, <code>^</code> means centre, <code>&lt;</code> means left, <code>&gt;</code> means right. <code>f"{s:^{w}}"</code> centres <code>s</code> in width <code>w</code>.</p>
</div>

<div class="pb-card">
<div class="pb-card-num">03</div>
<h4>Stars formula: 2i − 1</h4>
<p>Odd-number sequences (1, 3, 5, …) always follow <code>2i - 1</code>. Recognising this saves you re-deriving it in every pattern problem.</p>
