???+ abstract "⚡ Quick Reference"
    **Function:** `mm_dd_yy_to_yy_dd_mm(date: str) -> str`

    **Core idea:** split on `-`, reorder the three parts, rejoin.

    ```python
    def mm_dd_yy_to_yy_dd_mm(date: str) -> str:
        mm, dd, yy = date.split("-")
        return f"{yy}-{dd}-{mm}"
    ```

    **Key rules:**
    - Input: `"mm-dd-yy"` → Output: `"yy-dd-mm"`
    - Only the month and year swap positions; day stays in the middle
    - `split("-")` gives `[mm, dd, yy]` - unpack directly

---

## Problem Statement

!!! note "Problem"
    Write a function `mm_dd_yy_to_yy_dd_mm(date)` that converts a date string from `mm-dd-yy` to `yy-dd-mm` format.

**Example:**

<div class="pb-io">
<div class="pb-io-block input">
<span class="pb-io-label">Input</span>
<pre><code>"12-25-21"</code></pre>
</div>
<div class="pb-io-block output">
<span class="pb-io-label">Output</span>
<pre><code>"21-25-12"</code></pre>
</div>
</div>

`mm=12`, `dd=25`, `yy=21` → reassemble as `yy-dd-mm` = `"21-25-12"` ✓

## Solution approaches

=== "Pythonic (split + f-string)"

    ```python
    def mm_dd_yy_to_yy_dd_mm(date: str) -> str:
        mm, dd, yy = date.split("-")
        return f"{yy}-{dd}-{mm}"
    ```

=== "Using slicing"

    ```python
    def mm_dd_yy_to_yy_dd_mm(date: str) -> str:
        mm = date[:2]
        dd = date[3:5]
        yy = date[6:]
        return f"{yy}-{dd}-{mm}"
    ```

    Fixed positions: `mm` at 0-1, `dd` at 3-4, `yy` at 6-7. Direct slicing without splitting.

=== "One-liner with split"

    ```python
    def mm_dd_yy_to_yy_dd_mm(date: str) -> str:
        p = date.split("-")
        return "-".join([p[2], p[1], p[0]])
    ```

    Split into a list, reorder by index, rejoin. `[p[2], p[1], p[0]]` reverses the first and last parts.

=== "Using lambda"

    ```python
    mm_dd_yy_to_yy_dd_mm = lambda date: "-".join(date.split("-")[::-1][::-1][0:1] + date.split("-")[1:2] + date.split("-")[::-1][:1])
    ```

    Simpler lambda:

    ```python
    mm_dd_yy_to_yy_dd_mm = lambda d: (lambda p: f"{p[2]}-{p[1]}-{p[0]}")(d.split("-"))
    ```

    Split in an inner lambda, reassemble in the outer expression.

## Key takeaways

<div class="pb-cards">

<div class="pb-card">
<div class="pb-card-num">01</div>
<h4>split + unpack for structured strings</h4>
<p><code>mm, dd, yy = date.split("-")</code> splits and names the parts in one line. Unpacking directly is cleaner than indexing into the list.</p>
</div>

<div class="pb-card">
<div class="pb-card-num">02</div>
<h4>Only month and year swap</h4>
<p>The day stays in position 2 of the output. Only <code>mm</code> and <code>yy</code> exchange places. Re-read the format carefully - it's easy to miss that <code>dd</code> is unchanged.</p>
</div>

<div class="pb-card">
<div class="pb-card-num">03</div>
<h4>Slicing vs splitting</h4>
<p>Both work for fixed-format strings. <code>split("-")</code> is more readable when the separator is meaningful. Direct slicing (<code>date[:2]</code>) is useful when you know exact character positions.</p>
</div>
