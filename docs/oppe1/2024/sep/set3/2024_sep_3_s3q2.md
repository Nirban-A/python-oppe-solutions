???+ abstract "⚡ Quick Reference"
    **Type:** Full I/O - pattern printing

    **Core idea:** each row has `|` on both sides with `\` moving one step right each row. Row `i` has `i-1` spaces before `\` and `n-i` spaces after.

    ```python
    n = int(input())
    for i in range(1, n + 1):
        print("|" + " " * (i - 1) + "\\" + " " * (n - i) + "|")
    ```

    **Key rules:**
    - `|` on both sides of every row
    - `\` at position `i` within the inner area (1-indexed)
    - Leading spaces before `\`: `i - 1`
    - Trailing spaces after `\` (before closing `|`): `n - i`
    - No trailing spaces after the final `|`

---

## Problem Statement

!!! note "Problem (I/O type)"
    Given `n`, print an N-shaped pattern. `|` appears on both sides of every row, and `\` moves diagonally from top-left to bottom-right.

## Deriving the pattern

For `n=3`, the inner area between the `|` bars has width `n`:

```
Row 1:  |  \  ·  ·  |     leading=0, trailing=2
Row 2:  |  ·  \  ·  |     leading=1, trailing=1
Row 3:  |  ·  ·  \  |     leading=2, trailing=0
```

| Row `i` | Leading spaces | Trailing spaces |
|---------|---------------|----------------|
| 1 | `i-1 = 0` | `n-i = 2` |
| 2 | `i-1 = 1` | `n-i = 1` |
| 3 | `i-1 = 2` | `n-i = 0` |

Row structure: `|` + `(i-1)` spaces + `\` + `(n-i)` spaces + `|`

The trailing spaces (between `\` and `|`) are inside the pattern - they're needed to keep the right `|` properly aligned.

!!! insight "No trailing spaces after the closing |"
    The problem says no spaces to the right of the pattern. The spaces inside the bars are part of the pattern. The closing `|` is always the last character - nothing follows it.

## Tracing n = 3

| i | `i-1` spaces | `\` | `n-i` spaces | Row |
|---|-------------|-----|-------------|-----|
| 1 | `` | `\` | `··` | `|\··|` |
| 2 | `·` | `\` | `·` | `|·\·|` |
| 3 | `··` | `\` | `` | `|··\|` |

*(dots = spaces)*

## Solution approaches

=== "Pythonic (f-string)"

    ```python
    n = int(input())
    for i in range(1, n + 1):
        print(f"|{' '*(i-1)}\\{' '*(n-i)}|")
    ```

=== "Explanatory"

    ```python
    n = int(input())
    for i in range(1, n + 1):
        leading  = " " * (i - 1)
        trailing = " " * (n - i)
        print("|" + leading + "\\" + trailing + "|")
    ```

=== "Using str.center on inner content"

    ```python
    n = int(input())
    for i in range(1, n + 1):
        # Place \ at position i-1 (0-indexed) in a field of width n
        inner = " " * (i - 1) + "\\" + " " * (n - i)
        print("|" + inner + "|")
    ```

=== "Using lambda"

    ```python
    n = int(input())
    row = lambda i: "|" + " " * (i-1) + "\\" + " " * (n-i) + "|"
    for i in range(1, n+1):
        print(row(i))
    ```

---

## Edge case: n = 1

`i=1`: leading=0, trailing=0 → `|` + `` + `\` + `` + `|` = `|\|` ✓

## Key takeaways

<div class="pb-cards">

<div class="pb-card">
<div class="pb-card-num">01</div>
<h4>Diagonal \ at row i: leading = i-1</h4>
<p>The backslash moves one step right per row. At row <code>i</code> (1-indexed), it has <code>i-1</code> spaces before it and <code>n-i</code> spaces after - both decreasing and increasing symmetrically.</p>
</div>

<div class="pb-card">
<div class="pb-card-num">02</div>
<h4>Spaces inside bars are part of the pattern</h4>
<p>The trailing spaces between <code>\</code> and the closing <code>|</code> keep the right bar aligned. They're inside the pattern, not trailing whitespace - so they must stay.</p>
</div>

<div class="pb-card">
<div class="pb-card-num">03</div>
<h4>Always escape backslash: \\</h4>
<p>In Python strings, <code>\</code> is an escape character. Write <code>"\\"</code> to get a literal backslash in output. In f-strings, place <code>"\\"</code> outside the braces.</p>
</div>
