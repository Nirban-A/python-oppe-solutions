???+ abstract "⚡ Quick Reference"
    **Function:** `increment_value_with_max_limit(d, key, inc, limit) -> None`

    **Core idea:** add `inc` to `d[key]`, cap at `limit` using `min()`.

    ```python
    def increment_value_with_max_limit(d: dict, key: str, inc: int, limit: int):
        d[key] = min(d[key] + inc, limit)
    ```

    **Key rules:**
    - Modifies `d` in-place - returns `None`
    - `min(current + inc, limit)` caps the result automatically
    - Works for any increment, including when result is below limit

---

## Problem Statement

!!! note "Problem"
    Write a function `increment_value_with_max_limit(d, key, inc, limit)` that increments `d[key]` by `inc`, but caps it at `limit`. Modifies in-place, returns `None`.

**Examples:**

## Solution approaches

=== "Pythonic (min)"

    ```python
    def increment_value_with_max_limit(d: dict, key: str, inc: int, limit: int):
        d[key] = min(d[key] + inc, limit)
    ```

=== "Explanatory (if-else)"

    ```python
    def increment_value_with_max_limit(d: dict, key: str, inc: int, limit: int):
        new_value = d[key] + inc
        if new_value > limit:
            d[key] = limit
        else:
            d[key] = new_value
    ```

=== "Using lambda (assignment expression)"

    ```python
    def increment_value_with_max_limit(d: dict, key: str, inc: int, limit: int):
        d.__setitem__(key, min(d[key] + inc, limit))
    ```

    `dict.__setitem__(key, value)` is the explicit form of `d[key] = value`. Used when you need assignment in a functional expression.

## Key takeaways

<div class="pb-cards">

<div class="pb-card">
<div class="pb-card-num">01</div>
<h4>min() as a natural cap</h4>
<p><code>min(value, ceiling)</code> enforces an upper bound. The mirror - <code>max(value, floor)</code> - enforces a lower bound. Combine: <code>min(max(v, lo), hi)</code> clamps to a range.</p>
</div>

<div class="pb-card">
<div class="pb-card-num">02</div>
<h4>In-place dict modification</h4>
<p><code>d[key] = new_value</code> modifies the dict the caller passed in. No return needed. Always check whether a problem wants in-place modification or a new return value.</p>
</div>

<div class="pb-card">
<div class="pb-card-num">03</div>
<h4>Two-step: compute then cap</h4>
<p>Think of it as two separate operations: <code>d[key] + inc</code> (increment), then <code>min(..., limit)</code> (cap). Naming them separately in the explanatory approach makes debugging easier.</p>
</div>
