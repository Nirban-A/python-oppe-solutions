???+ abstract "⚡ Quick Reference"
    **Type:** Full I/O problem (no function — write complete code with `input()` / `print()`)

    **Core idea:** read all lines, loop in steps of 2, swap each pair, reverse the first of each pair (which lands at an odd output position).

    ```python
    n = int(input())
    lines = [input() for _ in range(n)]
    for i in range(0, n, 2):
        if i + 1 < n:
            print(lines[i + 1][::-1])   # odd output pos → reversed
            print(lines[i])              # even output pos → as-is
        else:
            print(lines[i][::-1])        # lone last line → reversed
    ```

    **Key rules:**
    - Read **all** lines first, then process (can't process on-the-fly)
    - After swapping, **output positions 1, 3, 5… are reversed**
    - A lone unpaired last line is always at an odd position → reverse it
    - `[::-1]` reverses any string in Python

## Problem Statement

!!! note "Problem (I/O type)"
    Given `n` lines of input, modify them as follows:

    1. **Swap** every two consecutive lines (line 1 ↔ line 2, line 3 ↔ line 4, …)
    2. **Reverse** every odd-numbered line **in the output sequence** (positions 1, 3, 5, …)
    3. If `n` is odd, the last line is unpaired — it is still reversed.

    **Input:** First line is `n`; next `n` lines are text strings.
    **Output:** Print the `n` transformed lines.

**Example:**

## Understanding what the rules actually mean

It's easy to misread this problem. Let's be precise.

**After swapping pair (apple, orange):**

```
Original order:   apple [pos 1]   orange [pos 2]
After swap:       orange [pos 1]  apple [pos 2]
```

Output position 1 is odd → `orange` gets reversed → `egnaro`.  
Output position 2 is even → `apple` stays → `apple`.

This means **inside every pair, the second original line comes out first and reversed; the first original line comes out second unchanged.**

!!! insight "Key insight"
    You don't need to track output position with a counter. Within each pair `(lines[i], lines[i+1])`:

    - Print `lines[i+1][::-1]` — swapped-in line, at odd position → reversed
    - Print `lines[i]`         — original first line, at even position → unchanged

## Processing the full example

Original list (0-indexed): `["apple", "orange", "pineapple", "mango", "kiwi"]`

| `i` | Pair | Out[pos] | Value printed |
|-----|------|-----------|--------------|
| 0 | (apple, orange) | Out[1] — odd | `orange[::-1]` = **egnaro** |
| 0 | (apple, orange) | Out[2] — even | `apple` |
| 2 | (pineapple, mango) | Out[3] — odd | `mango[::-1]` = **ognam** |
| 2 | (pineapple, mango) | Out[4] — even | `pineapple` |
| 4 | (kiwi — lone) | Out[5] — odd | `kiwi[::-1]` = **iwik** |

## Solution approaches

=== "Explanatory (direct print)"

    ```python
    n = int(input())
    lines = [input() for _ in range(n)]

    for i in range(0, n, 2):
        if i + 1 < n:
            # Pair exists: swap and print
            print(lines[i + 1][::-1])   # swapped-in → odd pos → reversed
            print(lines[i])              # stays put  → even pos → unchanged
        else:
            # Lone last line: odd position → reversed
            print(lines[i][::-1])
    ```

    - `range(0, n, 2)` steps through indices 0, 2, 4, … — one step per pair
    - `i + 1 < n` guards against index-out-of-range when `n` is odd

=== "Build result list"

    ```python
    n = int(input())
    lines = [input() for _ in range(n)]
    result = []

    for i in range(0, n, 2):
        # First output: swapped-in line (odd pos → reversed)
        result.append(lines[i + 1][::-1] if i + 1 < n else lines[i][::-1])
        # Second output: original first line (even pos → unchanged)
        if i + 1 < n:
            result.append(lines[i])

    print('\n'.join(result))
    ```

    Collects everything into a list, then prints in one call. Easier to test/debug.

=== "zip_longest (advanced)"

    ```python
    from itertools import zip_longest

    n = int(input())
    lines = [input() for _ in range(n)]

    # Split into even-indexed and odd-indexed sublists
    evens = lines[0::2]
    odds  = lines[1::2]

    for a, b in zip_longest(evens, odds):
        if b is not None:
            print(b[::-1])   # swapped-in → reversed
            print(a)
        else:
            print(a[::-1])   # lone last → reversed
    ```

    `zip_longest` pairs up the two sublists, filling `None` when `odds` runs short (i.e. `n` is odd). Very clean for interviews.

---

## Why you must read all lines first

You might wonder: why not process line by line as they come in?

The problem requires *swapping pairs*, which means to print line 2 before line 1, you need to have *seen* line 2 before printing anything. You can't read line 1, print it, then swap — you have to buffer at least 2 lines at a time. Reading all lines into a list upfront is the simplest way to handle this.
