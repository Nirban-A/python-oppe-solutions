???+ abstract "⚡ Quick Reference"
    **Function:** `within_and_has_double_quotes(s: str) -> bool`

    **Core idea:** check that first and last chars are `"`, and that `"` appears somewhere in the middle.

    ```python
    def within_and_has_double_quotes(s: str) -> bool:
        return (len(s) >= 3 and
                s[0] == '"' and
                s[-1] == '"' and
                '"' in s[1:-1])
    ```

    **Key rules:**
    - `s[0] == '"'` and `s[-1] == '"'` - outer quotes
    - `'"' in s[1:-1]` - at least one `"` in the interior
    - Minimum length 3: at least `"` + one inner `"` + `"` (e.g. `"""`)

---

## Problem Statement

!!! note "Problem"
    Write a function `within_and_has_double_quotes(s)` that returns `True` if `s` starts and ends with a double quote **and** contains at least one double quote in the interior (excluding the first and last characters).

## Understanding the problem

Three conditions, all must be true:

| Condition | Check |
|-----------|-------|
| Starts with `"` | `s[0] == '"'` |
| Ends with `"` | `s[-1] == '"'` |
| Has `"` inside | `'"' in s[1:-1]` |

`s[1:-1]` slices out everything except the first and last character - the interior. If a `"` is found there, the third condition passes.

!!! insight "Why check len(s) >= 3?"
    If `s` has fewer than 3 characters (e.g. `""` with length 2), `s[1:-1]` is an empty string - the interior check would always fail. The length guard avoids a false edge case and is also good defensive coding.

## Tracing all examples

| Input | `s[0]=='"'` | `s[-1]=='"'` | `'"' in s[1:-1]` | Result |
|-------|------------|-------------|-----------------|--------|
| `"abcd"efgh"` | ✅ | ✅ | ✅ (`"` at pos 5) | `True` |
| `'abcd"efgh"'` | ❌ (starts with `'`) | - | - | `False` |
| `"'abcd'efgh'"` | ❌ (starts with `"` - wait, let me re-read) | | | |

Re-reading example 3: `"'abcd'efgh'"` - in Python this is the string `'abcd'efgh'`. It starts with `'` not `"`. So `s[0] == '"'` is `False` → `False` 

| `"abcdefgh"` | ✅ | ✅ | ❌ (no `"` inside `abcdefgh`) | `False` |

## Solution approaches

=== "Pythonic (direct return)"

    ```python
    def within_and_has_double_quotes(s: str) -> bool:
        return (len(s) >= 3 and
                s[0] == '"' and
                s[-1] == '"' and
                '"' in s[1:-1])
    ```

=== "Explanatory (named conditions)"

    ```python
    def within_and_has_double_quotes(s: str) -> bool:
        if len(s) < 3:
            return False
        starts_with_quote = s[0] == '"'
        ends_with_quote   = s[-1] == '"'
        has_inner_quote   = '"' in s[1:-1]
        return starts_with_quote and ends_with_quote and has_inner_quote
    ```

=== "Using startswith/endswith"

    ```python
    def within_and_has_double_quotes(s: str) -> bool:
        return (len(s) >= 3 and
                s.startswith('"') and
                s.endswith('"') and
                '"' in s[1:-1])
    ```

    `str.startswith()` and `str.endswith()` are more readable than indexing for boundary checks - especially when checking multi-character prefixes/suffixes.

=== "Using lambda"

    ```python
    within_and_has_double_quotes = lambda s: (
        len(s) >= 3 and s[0] == '"' and s[-1] == '"' and '"' in s[1:-1]
    )
    ```

## Key takeaways

<div class="pb-cards">

<div class="pb-card">
<div class="pb-card-num">01</div>
<h4>s[1:-1] for the interior</h4>
<p>Slicing with <code>s[1:-1]</code> strips the first and last characters, giving only the interior. Use <code>in</code> to check membership - clean and readable.</p>
</div>

<div class="pb-card">
<div class="pb-card-num">02</div>
<h4>startswith() and endswith()</h4>
<p>More readable than <code>s[0] == x</code> for boundary checks, and handle multi-character prefixes/suffixes cleanly. Good habit for any "starts/ends with" condition.</p>
</div>

<div class="pb-card">
<div class="pb-card-num">03</div>
<h4>Length guard before slicing</h4>
<p>Always guard minimum length before accessing specific indices or slices. <code>len(s) >= 3</code> ensures <code>s[0]</code>, <code>s[-1]</code>, and <code>s[1:-1]</code> all behave correctly.</p>
</div>
