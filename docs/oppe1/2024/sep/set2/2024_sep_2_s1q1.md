???+ abstract "⚡ Quick Reference"
    **Function:** `is_positive_odd_or_negative_even(n: int) -> bool`

    **Core idea:** two separate conditions - either must be true.

    ```python
    def is_positive_odd_or_negative_even(n: int) -> bool:
        return (n > 0 and n % 2 != 0) or (n < 0 and n % 2 == 0)
    ```

    **Key rules:**
    - Positive odd: `n > 0 and n % 2 != 0`
    - Negative even: `n < 0 and n % 2 == 0`
    - `0` is neither positive nor negative → `False`

---

## Problem Statement

!!! note "Problem"

!!! insight "n % 2 for negative numbers"
    In Python, `n % 2` for negative numbers: `-4 % 2 = 0` (even), `-3 % 2 = 1` (odd). Python's modulo always returns a non-negative result when the divisor is positive, so `n % 2 == 0` correctly identifies even numbers regardless of sign.

## Solution approaches

=== "Pythonic (direct return)"

    ```python
    def is_positive_odd_or_negative_even(n: int) -> bool:
        return (n > 0 and n % 2 != 0) or (n < 0 and n % 2 == 0)
    ```

=== "Explanatory (named conditions)"

    ```python
    def is_positive_odd_or_negative_even(n: int) -> bool:
        is_positive_odd  = n > 0 and n % 2 != 0
        is_negative_even = n < 0 and n % 2 == 0
        return is_positive_odd or is_negative_even
    ```

=== "Using sign and parity"

    ```python
    def is_positive_odd_or_negative_even(n: int) -> bool:
        if n == 0:
            return False
        sign   = 1 if n > 0 else -1
        parity = n % 2   # 0 = even, 1 = odd
        # positive odd: sign=1, parity=1
        # negative even: sign=-1, parity=0
        return (sign == 1 and parity == 1) or (sign == -1 and parity == 0)
    ```

=== "Using lambda"

    ```python
    is_positive_odd_or_negative_even = lambda n: (
        (n > 0 and n % 2 != 0) or (n < 0 and n % 2 == 0)
    )
    ```

## Key takeaways

<div class="pb-cards">

<div class="pb-card">
<div class="pb-card-num">01</div>
<h4>Two compound conditions joined by or</h4>
<p>Each branch uses <code>and</code> to combine sign and parity checks. The two branches are joined by <code>or</code> - either qualifying is enough to return <code>True</code>.</p>
</div>

<div class="pb-card">
<div class="pb-card-num">02</div>
<h4>Zero is neither positive nor negative</h4>
<p><code>0 > 0</code> is <code>False</code> and <code>0 < 0</code> is <code>False</code> - so <code>0</code> fails both branches automatically. No special case needed.</p>
</div>

<div class="pb-card">
<div class="pb-card-num">03</div>
<h4>Python % always returns non-negative</h4>
<p>In Python, <code>-4 % 2 == 0</code> and <code>-3 % 2 == 1</code>. The modulo result is always between 0 and divisor-1, so <code>n % 2 == 0</code> reliably tests evenness for negative numbers too.</p>
</div>
