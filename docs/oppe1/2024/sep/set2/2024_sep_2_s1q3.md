???+ abstract "⚡ Quick Reference"
    **Function:** `replace_middle_with_n_times_middle(t: tuple, n: int) -> tuple`

    **Core idea:** find the middle index, slice around it, insert `n` copies of the middle element.

    ```python
    def replace_middle_with_n_times_middle(t: tuple, n: int) -> tuple:
        mid = len(t) // 2
        return t[:mid] + (t[mid],) * n + t[mid+1:]
    ```

    **Key rules:**
    - Input always has odd length → middle index = `len(t) // 2`
    - `(t[mid],) * n` creates a tuple of `n` copies
    - Concatenate: before + copies + after

---

## Problem Statement

!!! note "Problem"

## Understanding the problem

For `t = (1, 2, 3, 4, 5)`:
```
len(t) = 5  →  mid = 5 // 2 = 2
t[mid] = t[2] = 3

t[:2]      = (1, 2)
(3,) * 5   = (3, 3, 3, 3, 3)
t[3:]      = (4, 5)

Result: (1, 2) + (3,3,3,3,3) + (4, 5) = (1, 2, 3, 3, 3, 3, 3, 4, 5) ✓
```

!!! insight "(element,) * n - single-element tuple repeated"
    To create a tuple of `n` copies of a value: `(value,) * n`. The trailing comma after `value` makes it a tuple (not parentheses). Without the comma: `(value) * n` is just `value * n` - a number multiplied.

---

## Tracing both examples

**`t=(1,2,3,4,5)`, `n=5`:**
```
mid = 2,  t[mid] = 3
t[:2] + (3,)*5 + t[3:]
= (1,2) + (3,3,3,3,3) + (4,5)
= (1, 2, 3, 3, 3, 3, 3, 4, 5) ✓
```

**`t=(1,2,3)`, `n=4`:**
```
mid = 1,  t[mid] = 2
t[:1] + (2,)*4 + t[2:]
= (1,) + (2,2,2,2) + (3,)
= (1, 2, 2, 2, 2, 3) ✓
```

## Solution approaches

=== "Pythonic (slice + tuple multiply)"

    ```python
    def replace_middle_with_n_times_middle(t: tuple, n: int) -> tuple:
        mid = len(t) // 2
        return t[:mid] + (t[mid],) * n + t[mid+1:]
    ```

=== "Explanatory (named parts)"

    ```python
    def replace_middle_with_n_times_middle(t: tuple, n: int) -> tuple:
        mid     = len(t) // 2
        before  = t[:mid]
        copies  = (t[mid],) * n
        after   = t[mid+1:]
        return before + copies + after
    ```

=== "Using list then convert"

    ```python
    def replace_middle_with_n_times_middle(t: tuple, n: int) -> tuple:
        l = list(t)
        mid = len(l) // 2
        l[mid:mid+1] = [l[mid]] * n    # replace one element with n copies
        return tuple(l)
    ```

    Convert to list (mutable), use slice assignment to replace the middle element with `n` copies, convert back. The slice `l[mid:mid+1] = [...]` replaces exactly one element.

=== "Using lambda"

    ```python
    replace_middle_with_n_times_middle = lambda t, n: (
        t[:len(t)//2] + (t[len(t)//2],) * n + t[len(t)//2+1:]
    )
    ```

## Key takeaways

<div class="pb-cards">

<div class="pb-card">
<div class="pb-card-num">01</div>
<h4>(value,) * n - tuple repetition</h4>
<p>The trailing comma makes <code>(value,)</code> a single-element tuple. Multiplying by <code>n</code> gives <code>n</code> copies. Without the comma, <code>(value) * n</code> is arithmetic, not a tuple.</p>
</div>

<div class="pb-card">
<div class="pb-card-num">02</div>
<h4>len(t) // 2 for odd-length middle</h4>
<p>For any odd-length sequence, integer division by 2 gives the exact middle index: length 3 → index 1, length 5 → index 2, length 7 → index 3.</p>
</div>

<div class="pb-card">
<div class="pb-card-num">03</div>
<h4>l[mid:mid+1] = [...] for in-place replacement</h4>
<p>Slice assignment <code>l[i:i+1] = new_elements</code> replaces exactly one element with any number of elements - a flexible in-place substitution pattern.</p>
</div>
