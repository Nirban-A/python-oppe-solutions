???+ abstract "⚡ Quick Reference"
    **Function:** `move_even_indices_to_end_reversed(t: tuple) -> tuple`

    **Core idea:** separate odd-indexed elements (keep order) and even-indexed elements (reverse), then concatenate.

    ```python
    def move_even_indices_to_end_reversed(t):
        odd_indexed  = t[1::2]          # elements at indices 1, 3, 5, …
        even_indexed = t[0::2]          # elements at indices 0, 2, 4, …
        return odd_indexed + even_indexed[::-1]
    ```

    **Key rules:**
    - `t[1::2]` → odd-indexed elements in original order
    - `t[0::2]` → even-indexed elements in original order
    - `[::-1]` reverses the even-indexed elements
    - Concatenate: odd first, reversed evens last

---

## Problem Statement

!!! note "Problem"
    Write a function `move_even_indices_to_end_reversed(t)` that returns a new tuple with:
    - Odd-indexed elements in their original relative order
    - Even-indexed elements appended at the end in **reversed** order

## Understanding the problem

For `t = (10, 20, 30, 40, 50)`:

```
Index:   0    1    2    3    4
Value:  10   20   30   40   50
        ↑         ↑         ↑   ← even indices (0, 2, 4)
             ↑         ↑        ← odd indices  (1, 3)
```

Step 1 - collect odd-indexed elements (keep order):
```
indices 1, 3  →  (20, 40)
```

Step 2 - collect even-indexed elements and reverse:
```
indices 0, 2, 4  →  (10, 30, 50)  →  reversed: (50, 30, 10)
```

Step 3 - concatenate:
```
(20, 40) + (50, 30, 10)  =  (20, 40, 50, 30, 10) 
```

!!! insight "Step slicing: t[start::step]"
    - `t[0::2]` starts at index 0 and takes every 2nd element → even indices
    - `t[1::2]` starts at index 1 and takes every 2nd element → odd indices

    These are the cleanest way to split a sequence by index parity.

## Solution approaches

=== "Pythonic (step slicing)"

    ```python
    def move_even_indices_to_end_reversed(t):
        odd_indexed  = t[1::2]
        even_indexed = t[0::2]
        return odd_indexed + even_indexed[::-1]
    ```

=== "Explanatory (loop)"

    ```python
    def move_even_indices_to_end_reversed(t):
        odd_indexed  = []
        even_indexed = []
        for i, val in enumerate(t):
            if i % 2 == 0:
                even_indexed.append(val)
            else:
                odd_indexed.append(val)
        return tuple(odd_indexed) + tuple(reversed(even_indexed))
    ```

=== "Using list comprehensions"

    ```python
    def move_even_indices_to_end_reversed(t):
        odd_indexed  = tuple(t[i] for i in range(1, len(t), 2))
        even_indexed = tuple(t[i] for i in range(0, len(t), 2))
        return odd_indexed + even_indexed[::-1]
    ```

=== "Using filter + lambda"

    ```python
    def move_even_indices_to_end_reversed(t):
        indexed = list(enumerate(t))
        odd_vals  = tuple(v for i, v in indexed if i % 2 != 0)
        even_vals = tuple(v for i, v in indexed if i % 2 == 0)
        return odd_vals + even_vals[::-1]
    ```

    `enumerate(t)` pairs each value with its index. Filter by parity to separate the two groups, then reverse the evens.

=== "Using lambda"

    ```python
    move_even_indices_to_end_reversed = lambda t: t[1::2] + t[0::2][::-1]
    ```

    Entire function as a one-liner lambda. Step slicing makes it clean enough to fit on one line.

## Key takeaways

<div class="pb-cards">

<div class="pb-card">
<div class="pb-card-num">01</div>
<h4>t[0::2] and t[1::2] split by parity</h4>
<p><code>t[0::2]</code> picks every element at an even index; <code>t[1::2]</code> picks every element at an odd index. Together they partition the entire tuple without any loop or condition.</p>
</div>

<div class="pb-card">
<div class="pb-card-num">02</div>
<h4>[::-1] reverses a tuple</h4>
<p>The step-slice <code>[::-1]</code> works on tuples just like on lists and strings - it returns a reversed copy without modifying the original.</p>
</div>

<div class="pb-card">
<div class="pb-card-num">03</div>
<h4>Odd elements keep their order</h4>
<p>Only the even-indexed elements are reversed. Odd-indexed elements appear at the front in their original relative order. Read the problem carefully - it's easy to reverse the wrong group.</p>
</div>
