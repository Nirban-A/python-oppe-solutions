???+ abstract "⚡ Quick Reference"
    **Function:** `is_repeated_thrice(s: str) -> bool`

    **Core idea:** length must be divisible by 3, and all three equal-length parts must be identical.

    ```python
    def is_repeated_thrice(s: str) -> bool:
        if len(s) % 3 != 0:
            return False
        part = len(s) // 3
        return s[:part] == s[part:2*part] == s[2*part:]
    ```

    **Key rules:**
    - `len(s) % 3 != 0` → immediately `False`
    - Split into three equal slices and compare all three
    - Chained `==` checks all three in one expression

---

## Problem Statement

!!! note "Problem"
    Write a function `is_repeated_thrice(s)` that returns `True` if `s` is formed by repeating the same substring exactly three times - i.e. its length is divisible by 3 and all three equal parts are identical.

## Understanding the problem

Two conditions must both be true:

1. `len(s) % 3 == 0` - length is divisible by 3
2. `s[:n] == s[n:2n] == s[2n:]` - all three parts are identical (where `n = len(s) // 3`)

```
"abcabcabc"  →  n=3
Part 1: s[0:3] = "abc"
Part 2: s[3:6] = "abc"
Part 3: s[6:9] = "abc"
All equal → True 

"hello"  →  len=5, 5 % 3 = 2 ≠ 0 → False 
```

!!! insight "Chained comparison: a == b == c"
    Python allows chaining equality checks: `a == b == c` is equivalent to `a == b and b == c`. This checks all three parts in one clean expression.

## Solution approaches

=== "Pythonic (slicing + chained ==)"

    ```python
    def is_repeated_thrice(s: str) -> bool:
        if len(s) % 3 != 0:
            return False
        n = len(s) // 3
        return s[:n] == s[n:2*n] == s[2*n:]
    ```

=== "Explanatory (named parts)"

    ```python
    def is_repeated_thrice(s: str) -> bool:
        if len(s) % 3 != 0:
            return False
        n = len(s) // 3
        part1 = s[:n]
        part2 = s[n:2*n]
        part3 = s[2*n:]
        return part1 == part2 == part3
    ```

=== "Using string multiplication"

    ```python
    def is_repeated_thrice(s: str) -> bool:
        if len(s) % 3 != 0:
            return False
        part = s[:len(s) // 3]
        return part * 3 == s
    ```

    Extract the first third, multiply it by 3, and check if it reconstructs `s`. Clean and readable - reads like the problem statement itself.

=== "Using all() + slicing"

    ```python
    def is_repeated_thrice(s: str) -> bool:
        if len(s) % 3 != 0:
            return False
        n = len(s) // 3
        parts = [s[i*n:(i+1)*n] for i in range(3)]
        return all(p == parts[0] for p in parts)
    ```

    Splits into three parts using a list comprehension, then checks all are equal to the first. Scales naturally if the problem were generalised to `k` repetitions.

=== "Using lambda"

    ```python
    is_repeated_thrice = lambda s: (
        len(s) % 3 == 0 and s[:len(s)//3] * 3 == s
    )
    ```

    Combines both checks in one expression using `and`. The `* 3` trick keeps it compact.

## Key takeaways

<div class="pb-cards">

<div class="pb-card">
<div class="pb-card-num">01</div>
<h4>part * 3 == s - the reconstruction trick</h4>
<p>If a string is truly made of three identical parts, repeating the first third three times must give back the original. <code>part * 3 == s</code> is the most readable way to express this.</p>
</div>

<div class="pb-card">
<div class="pb-card-num">02</div>
<h4>Chained equality: a == b == c</h4>
<p>Python's chained comparisons work for equality too. <code>s[:n] == s[n:2n] == s[2n:]</code> is cleaner than two separate <code>and</code> conditions.</p>
</div>

<div class="pb-card">
<div class="pb-card-num">03</div>
<h4>Check length first - short-circuit</h4>
<p>The length check is O(1); slicing and comparing is O(n). Always put the cheap condition first so the expensive comparison is skipped when the string length is wrong.</p>
</div>
