???+ abstract "⚡ Quick Reference"
    **Type:** Full I/O - string replacement with a running counter

    **Core idea:** for each line, replace every `[]` with `[n]` where `n` is a global counter incrementing left to right, top to bottom.

    ```python
    n = int(input())
    counter = 1
    for _ in range(n):
        line = input()
        result = []
        i = 0
        while i < len(line):
            if line[i:i+2] == "[]":
                result.append(f"[{counter}]")
                counter += 1
                i += 2
            else:
                result.append(line[i])
                i += 1
        print("".join(result))
    ```

    **Key rules:**
    - Replace `[]` (not `[anything]`) with `[counter]`
    - Counter is global across all lines - continues from where the previous line left off
    - All other characters remain unchanged

## Problem Statement

!!! note "Problem (I/O type)"
    Read `n` lines of text. Replace each `[]` with `[1]`, `[2]`, `[3]`, … in left-to-right, top-to-bottom order.

**Example:**

## Understanding the approach

The simplest approach: use `str.replace("[]", "[n]", 1)` in a loop - replace one `[]` at a time, incrementing the counter.

```python
line = "Programming[] is Good[]"
counter = 1

# First replacement:
line = line.replace("[]", f"[{counter}]", 1)  # "Programming[1] is Good[]"
counter = 2

# Second replacement:
line = line.replace("[]", f"[{counter}]", 1)  # "Programming[1] is Good[2]"
counter = 3
```

The `count=1` argument to `replace()` replaces only the **first** occurrence each call.

!!! insight "str.replace(old, new, count)"
    The third argument limits how many replacements are made. `replace("[]", "[n]", 1)` replaces only the first `[]` found. Calling it in a loop with an incrementing counter handles all occurrences sequentially.


Line 3: `"jumps[] over the lazy[] dog[]"` (counter starts at 5)
```
[5], [6], [7] → "jumps[5] over the lazy[6] dog[7]"
```

## Solution approaches

=== "Pythonic (replace in loop)"

    ```python
    n = int(input())
    counter = 1
    for _ in range(n):
        line = input()
        while "[]" in line:
            line = line.replace("[]", f"[{counter}]", 1)
            counter += 1
        print(line)
    ```

    Cleanest approach - `while "[]" in line` keeps replacing until none remain.

=== "Explanatory (character scan)"

    ```python
    n = int(input())
    counter = 1
    for _ in range(n):
        line = input()
        result = []
        i = 0
        while i < len(line):
            if line[i:i+2] == "[]":
                result.append(f"[{counter}]")
                counter += 1
                i += 2
            else:
                result.append(line[i])
                i += 1
        print("".join(result))
    ```

    Scans character by character - `[]` is a 2-char token, so skip 2 when matched.

=== "Using re.sub with a counter"

    ```python
    import re

    n = int(input())
    counter = [1]   # mutable to allow modification inside lambda

    def replace_bracket(match):
        result = f"[{counter[0]}]"
        counter[0] += 1
        return result

    lines = [input() for _ in range(n)]
    full_text = "\n".join(lines)
    output = re.sub(r'\[\]', replace_bracket, full_text)
    print(output)
    ```

    `re.sub` with a callable replacement - the function is called once per match, incrementing the counter each time. Processes all lines at once.

=== "Using lambda with itertools.count"

    ```python
    import re
    from itertools import count

    n = int(input())
    c = count(1)
    lines = "\n".join(input() for _ in range(n))
    print(re.sub(r'\[\]', lambda m: f"[{next(c)}]", lines))
    ```

    `itertools.count(1)` is an infinite counter. `next(c)` advances it each time the lambda is called by `re.sub`. Most compact approach - one line of processing after reading input.

## Key takeaways

<div class="pb-cards">

<div class="pb-card">
<div class="pb-card-num">01</div>
<h4>str.replace(old, new, 1) for one-at-a-time replacement</h4>
<p>The third argument <code>count=1</code> limits to the first occurrence. Calling it in a <code>while "[]" in line</code> loop replaces each bracket sequentially with the correct number.</p>
</div>

<div class="pb-card">
<div class="pb-card-num">02</div>
<h4>Global counter across lines</h4>
<p>The counter must persist across all lines - initialise it before the outer loop and never reset it between lines. Line 2 continues from where line 1 left off.</p>
</div>

<div class="pb-card">
<div class="pb-card-num">03</div>
<h4>re.sub with callable - powerful for sequential replacements</h4>
<p>When the replacement depends on state (like a counter), pass a function to <code>re.sub</code> instead of a string. The function is called once per match, enabling stateful replacements cleanly.</p>
</div>
