???+ abstract "⚡ Quick Reference"
    **Function:** `get_element_at_circular_index(lst: list, idx: int)`

    **Core idea:** `idx % len(lst)` wraps any index - positive or negative - into a valid position.

    ```python
    def get_element_at_circular_index(lst: list, idx: int):
        return lst[idx % len(lst)]
    ```

    **Key rules:**
    - `idx % len(lst)` handles both overflow and negative indices
    - Python's `%` always returns a non-negative result when divisor is positive
    - No bounds checking needed - modulo does it all

## Understanding circular indexing

Think of the list as a circle - after the last element, it wraps back to the first:

```
List: [1, 2, 3]  (length 3)

idx=0 → 0%3=0 → 1
idx=1 → 1%3=1 → 2
idx=2 → 2%3=2 → 3
idx=3 → 3%3=0 → 1  (wraps!)
idx=4 → 4%3=1 → 2  ✓
idx=5 → 5%3=2 → 3

idx=-1 → -1%3=2 → 3  (Python % always non-negative)
idx=-4 → -4%3=2 → 3
```

!!! insight "Python's % always returns non-negative"
    In Python, `-1 % 3 = 2` (not -1 as in some other languages). This means `idx % len(lst)` handles negative indices correctly without any special case - a single expression covers all situations.

## Tracing all examples

| `lst` | `idx` | `len(lst)` | `idx % len` | Element |
|-------|-------|-----------|-------------|---------|
| `[1,2,3]` | 4 | 3 | 4%3 = **1** | `2` ✓ |
| `[10,20,30,40]` | -1 | 4 | -1%4 = **3** | `40` ✓ |
| `["a","b","c"]` | 5 | 3 | 5%3 = **2** | `"c"` ✓ |
| `[7,8,9]` | 0 | 3 | 0%3 = **0** | `7` ✓ |

## Solution approaches

=== "Pythonic (one-liner)"

    ```python
    def get_element_at_circular_index(lst: list, idx: int):
        return lst[idx % len(lst)]
    ```

=== "Explanatory (named steps)"

    ```python
    def get_element_at_circular_index(lst: list, idx: int):
        circular_idx = idx % len(lst)   # always in range [0, len-1]
        return lst[circular_idx]
    ```

=== "Using lambda"

    ```python
    get_element_at_circular_index = lambda lst, idx: lst[idx % len(lst)]
    ```

## Key takeaways

<div class="pb-cards">

<div class="pb-card">
<div class="pb-card-num">01</div>
<h4>idx % len(lst) - one expression for all cases</h4>
<p>Modulo maps any integer index - however large or negative - into the valid range <code>[0, len-1]</code>. No <code>if</code> statements, no bounds checking needed.</p>
</div>

<div class="pb-card">
<div class="pb-card-num">02</div>
<h4>Python % is always non-negative (for positive divisor)</h4>
<p>Unlike C or Java, Python's <code>%</code> always returns a result with the same sign as the divisor. So <code>-1 % 4 == 3</code>, which is exactly the circular index for the last element.</p>
</div>

<div class="pb-card">
<div class="pb-card-num">03</div>
<h4>This is already how Python's negative indexing works</h4>
<p>Python's built-in <code>lst[-1]</code> uses the same modulo logic internally. The difference here is that <code>idx % len</code> also handles indices beyond the list length, which Python's normal indexing does not.</p>
</div>
