???+ abstract "⚡ Quick Reference"
    **Function:** `signed_abs_diff_of_middle_digits(n: int) -> int`

    **Core idea:** work on the digit string of `abs(n)`, find the middle, compute the result, then restore the sign.

    ```python
    def signed_abs_diff_of_middle_digits(n: int) -> int:
        sign   = -1 if n < 0 else 1
        digits = str(abs(n))
        length = len(digits)
        mid    = length // 2
        if length % 2 == 1:          # odd digits → single middle
            result = int(digits[mid])
        else:                         # even digits → abs diff of two middles
            result = abs(int(digits[mid - 1]) - int(digits[mid]))
        return sign * result
    ```

    **Key rules:**
    - Strip sign first: work on `abs(n)` digits
    - Odd length → middle digit at index `len // 2`
    - Even length → `abs(digits[mid-1] - digits[mid])` where `mid = len // 2`
    - Reapply sign at the end: `sign * result`

## Problem Statement

!!! note "Problem"

## Understanding the problem

**Step 1:** Extract digits from `abs(n)` as a string.

**Step 2:** Find the middle based on length:

```
"12345"  → length=5 (odd),  mid=2  → digits[2] = '3'        → result = 3
"987"    → length=3 (odd),  mid=1  → digits[1] = '8'        → result = 8
"9897"   → length=4 (even), mid=2  → |digits[1]-digits[2]|
                                    = |'8'-'9'| ... wait
```

For `-9897`: digits = `"9897"`, mid = 2
- `digits[mid-1]` = `digits[1]` = `'8'`
- `digits[mid]`   = `digits[2]` = `'9'`
- `abs(8 - 9)` = 1, sign = -1 → **-1** ✓

For `-1357`: digits = `"1357"`, mid = 2
- `digits[1]` = `'3'`, `digits[2]` = `'5'`
- `abs(3 - 5)` = 2, sign = -1 → **-2** ✓

**Step 3:** Apply sign of `n`.

## Solution approaches

=== "Pythonic"

    ```python
    def signed_abs_diff_of_middle_digits(n: int) -> int:
        sign   = -1 if n < 0 else 1
        digits = str(abs(n))
        length = len(digits)
        mid    = length // 2
        if length % 2 == 1:
            result = int(digits[mid])
        else:
            result = abs(int(digits[mid - 1]) - int(digits[mid]))
        return sign * result
    ```

=== "Explanatory (named variables)"

    ```python
    def signed_abs_diff_of_middle_digits(n: int) -> int:
        # Determine sign
        sign = -1 if n < 0 else 1

        # Work on digits of absolute value
        digits = str(abs(n))
        length = len(digits)
        mid    = length // 2

        if length % 2 == 1:
            # Odd length - single middle digit
            middle_digit = int(digits[mid])
            result = middle_digit
        else:
            # Even length - two middle digits
            left  = int(digits[mid - 1])
            right = int(digits[mid])
            result = abs(left - right)

        return sign * result
    ```

=== "Using lambda"

    ```python
    def signed_abs_diff_of_middle_digits(n: int) -> int:
        sign   = -1 if n < 0 else 1
        d      = str(abs(n))
        m      = len(d) // 2
        result = (int(d[m]) if len(d) % 2 == 1
                  else abs(int(d[m-1]) - int(d[m])))
        return sign * result
    ```

    Ternary expression picks between the two cases. Compact without sacrificing readability.

## Key takeaways

<div class="pb-cards">

<div class="pb-card">
<div class="pb-card-num">01</div>
<h4>Strip sign first, reapply at the end</h4>
<p>Work on <code>str(abs(n))</code> to avoid dealing with the minus sign in the digit string. Store the sign separately as <code>-1</code> or <code>+1</code> and multiply at the end.</p>
</div>

<div class="pb-card">
<div class="pb-card-num">02</div>
<h4>mid = len // 2 - works for both parities</h4>
<p>For odd length, <code>digits[mid]</code> is the exact centre. For even length, the two middle digits are at <code>digits[mid-1]</code> and <code>digits[mid]</code>. One formula covers both with the parity check.</p>
</div>

<div class="pb-card">
<div class="pb-card-num">03</div>
<h4>sign * 0 == 0 regardless of sign</h4>
<p>When the two middle digits are equal (like <code>-3333</code>), <code>abs(3-3) = 0</code> and <code>-1 * 0 = 0</code>. The sign doesn't matter for zero - both positive and negative give the same result.</p>
</div>
